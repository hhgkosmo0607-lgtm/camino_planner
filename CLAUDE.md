## 프로젝트

한국인 순례자를 위한 **카미노 데 산티아고 여정 관리 서비스**.
출국 90일 전 코스 선정부터 현지 도보, 귀국 후 기록까지를 한국어로 관리한다.

**여섯 축**: **A. 몸(부상 예방·대응)** / **B. 잠자리(숙소 예약 판단)** / **C. 길(경로 판단·내비게이션)** / **D. 다음(완주 후 다음 여정)** / **E. 가족(원격 안심)** / **F. 동행(매칭·정보 공유)**
**무게중심은 여전히 A축이다.** C·D·E·F는 A·B의 부속이 아니라 각자 다른 문제·고객군이지만, "부상이 완주를 막는다"는 근본 문제의 우선순위는 바뀌지 않는다. 축 정의 전체는 00 문서 2.1절, 수익화는 02 문서 1부 5장.

**F축(동행)은 2026-07 이전에는 명시적으로 제외돼 있었다** ("동행자: 불가", "SNS 피드형 커뮤니티 제외" — 네이버 카페 선점, 배터리·프라이버시 우려). 지금은 판단을 뒤집어 경쟁하기로 했지만, **원래 우려는 범위를 좁혀서 반영한다**: 전체 공개 피드가 아니라 opt-in 매칭이고, 실시간 위치 스트리밍이 아니라 마을 도착 시 체크인 신호다.
**목표**: 한국인 순례자의 **도달률 85%**
**한 문장 (2026-07 개정)**: **혼자 걷지만, 판단까지 혼자 하지 않게 돕는다.**
**차별점**: 데이터를 보여주지 않는다. **판단을 대신 내려준다.** 다만 사용자에게 보이는 표현은 "우리가 결정한다"가 아니라 "곁에서 돕는다"여야 한다 — 실질(판단 대행)과 표현(동반)을 구분한다. (기획서 1.1절 근거)

**웹과 앱은 순차(웹 먼저, 앱은 나중)가 아니라 병렬 트랙이다.** 장기적으로는 **앱이 더 중요한 채널**이며, 두 트랙에 같은 비중을 둔다.

**현재 실행 중: 웹 트랙 Phase 1 — 웹 계산기 + SEO.** 지금 웹만 코드로 나오는 이유는 우선순위가 아니라 **검증 순서** 때문이다 — 실측 도보와 경로·고도 데이터 검증(01·05 문서)이 끝나야 앱을 안정적으로 만들 수 있다. 검증이 끝나면 **앱 트랙은 웹의 특정 성과 지표(예: 트래픽) 달성을 기다리지 않고 병행 착수한다.** 앱도, DB도, 로그인도 아직 만들지 않은 것은 이 검증 단계이기 때문이다.

> 서비스명은 미확정이다. 브랜드명은 `config/brand.ts`에서만 관리하고 다른 곳에 하드코딩하지 않는다.

## 절대 규칙

### 1. 데이터를 지어내지 않는다 (최우선)

이 서비스는 순례자의 안전과 직결된다. **없는 알베르게를 만들어내거나, 확인되지 않은 거리·요금·전화번호·예약 가능 여부를 채워 넣으면 사람이 한데서 잔다.**

- 숙소 정보를 **추정으로 채우지 않는다.** 없으면 `null` 또는 `UNKNOWN`
- UI에서는 "정보 확인 중"으로 표시한다
- 샘플 데이터는 `source: 'PLACEHOLDER'`로 표시하고 **화면에도 샘플임을 노출**한다
- `data/towns.ts`의 거리·고도는 검증된 값이다. **임의로 수정하거나 보간하지 않는다**

### 2. 도메인 사실을 하드코딩하지 않는다

정책은 바뀐다. 2025년 초 포르투갈 길 친구회가 성수기 공립 알베르게 예약 허용을 제안했고, 2027년 성년을 앞두고 바뀔 수 있다.

- ❌ `if (albergue.type === 'MUNICIPAL') return '예약 불가'`
- ✅ `albergue.reservation` 필드를 읽는다

### 3. 고도는 반드시 profiles.ts를 경유한다 🔴

**마을 고도 차이로 오르막을 계산하면 최대 39% 틀린다.**

생장 → 론세스바예스는 마을 기준 +780m지만, 실제로는 레포에데르 고개(약 1,450m)를 넘으므로 누적 상승이 약 1,280m다. 카미노에서 부상이 가장 많은 날인데 "평범한 산길"로 판단된다.

**하강도 마찬가지로 중요하다.** 무릎 부상의 주원인은 오르막이 아니라 내리막이다. 엘 아세보 → 몰리나세카(−550m 급경사 돌길)는 `elevationGain` 기준으로 0이라 "쉬운 날"이 된다.

- ❌ `toTown.elevation - fromTown.elevation`
- ✅ `getProfile(fromId, toId).ascent` / `.descent`

`data/profiles.ts`는 `scripts/build-profiles.ts`가 생성한다.
경로는 OpenStreetMap(ODbL), 고도는 스페인 IGN MDT05(CC BY 4.0)에서 가져와 직접 계산한다.

- `source: 'ESTIMATED'` 인 동안은 **`injuryRiskScore`를 사용자에게 숫자로 노출하지 않는다**
- **좌표를 `profiles.ts`에 넣지 않는다.** 숫자만 남긴다 (ODbL 파생 DB 배포 회피)
- `data/geometry/` 는 gitignore 대상이다
- 출처 표시 의무: "경로 데이터 © OpenStreetMap contributors (ODbL)" / "고도 데이터 © Instituto Geográfico Nacional (CC BY 4.0)"

### 4. 프랑스 길은 하나의 선이 아니다 🔴

**11곳에서 갈라졌다 다시 만난다.** 현재 `towns.ts`는 단일 선형 배열이라 변형 경로가 없다.

- 경로는 `RouteFork` / `RouteVariant`로 관리한다
- **계절 폐쇄를 반드시 확인한다** — `closedFrom` / `closedTo`
- ❌ 모든 순례자가 같은 마을을 지난다고 가정하는 코드
- ✅ `variantId`를 `Stage`에 담아 어느 쪽을 걷는지 기록한다

| 대표 갈림길 | 선택지 |
|---|---|
| **생장피드포르** | **나폴레옹(고지)** vs **발카를로스(계곡)** |
| 레온 이후 | 비야당고스(차도) vs 비야르 데 마사리페 |
| 트리아카스텔라 | 산 실(짧음) vs 사모스(대수도원) |

⚠️ **나폴레옹 루트는 11월 1일~3월 31일 폐쇄된다.** 오리송 위로 대피소가 없다.
출발일이 폐쇄 기간이면 **후보에서 아예 뺀다.**

⚠️ **당일 안전 판단은 생장 순례자 사무소가 한다.** 우리는 사전 정보만 제공하고, 현지 안내를 따르라고 말한다.

### 5. 원어명을 지우면 버그다

마을과 숙소 이름은 **스페인어 원어를 반드시 함께 표시**한다. 순례자가 현지에서 화면을 보여줘야 하기 때문이다. 한글만 남기는 UI는 거절한다.

### 6. 계산 로직은 순수 함수 + 테스트

일정 분할, 거리, 고도, 위험 점수는 전부 `lib/planner/` 아래 순수 함수로 두고 단위 테스트를 붙인다.

- 컴포넌트 안에서 계산하지 않는다
- `fetch`, `window`, `localStorage` 참조 금지 (Phase 3에서 React Native로 이식)

### 7. SEO는 전략 그 자체다

검색이 이 서비스의 유일한 100% 관문이다.

- 모든 페이지 **서버 렌더**. JS를 꺼도 본문이 보인다
- `useEffect` 안에서 계산해서 렌더하지 않는다
- 페이지마다 `generateMetadata`로 고유 title (50~60자)
- 새 페이지를 만들면 sitemap에 반드시 포함

### 8. 무료이고 가입이 없다

- 핵심 기능에 결제벽·가입벽을 세우지 않는다
- 계획 상태는 **URL 쿼리스트링**에 담는다. localStorage 금지 (URL이 진실의 원천)
- 이메일은 선택 사항. 결과를 보려면 이메일을 내라는 구조 금지
- 체중·나이 같은 민감 정보를 필수 입력으로 만들지 않는다

### 9. 접근성 하한선

주 사용자층에 60대 이상이 상당수다.

- 본문 최소 17px, 주요 수치 18px 이상
- 명도 대비 4.5:1 이상
- 터치 타깃 최소 44×44px
- 키보드 포커스 링 제거 금지
- **인쇄 지원 필수** (`@media print`) — 비디지털층에 닿는 유일한 경로

### 10. 오프라인을 전제로 설계

Phase 3에서 React Native로 옮긴다. 메세타와 갈리시아 산간은 통신이 끊긴다.
**서버 API 호출로 일정을 계산하는 설계는 금지.**

### 11. 의료 정보는 넣지 않는다

통증 대처법, 약품명, 물집 처치 방법은 **정형외과·스포츠의학 자문 검수 전까지 앱에 넣지 않는다.**

### 12. 인증서를 흉내내지 않는다

완주 증명은 이미 공식 제도가 있다.

- **콤포스텔라** — 산티아고 대성당 발급
- **이중 순례자(Dual Pilgrim)** — 타나베시 관광청·산티아고 공동 운영

우리는 **안내와 기록만** 한다. 우리 이름으로 증서를 발급하거나 인증을 대행하지 않는다.
NFT·블록체인 기반 증서도 만들지 않는다 (→ 03 문서 6.2절).

완주 보상의 기본형은 **무료**다. 유료는 포토북처럼 실제 원가가 발생하는 것에 한정한다.
**"완주했는데 돈을 내야 결과를 준다"는 구조는 만들지 않는다.**

**⚠️ 90/10 원칙 (신설):** 보상을 나눌 때 비율이 아니라 **경계선**으로 판단한다.
- "이걸 안 사면 성취가 덜해 보이나?" → YES면 **무조건 무료** (콤포스텔라 인정, 도장·안개지도, 내 카미노 숫자, 이중 순례자 안내, 1년 후 편지)
- "이걸 사면 실물로 더 오래 간직·과시하나?" → YES면 **유료 가능** (포토북·포스터, 안개지도 포스터 색상 선택, 일회용 필름 카메라, 완주 기념 배지)
- 비율을 맞추려고 무료 항목을 깎지 않는다. 이 기준으로 걸러진 결과가 자연히 소수여야 한다

## 기술 스택 (웹 트랙, 현재 실행 단계)

```
--ink:     #12253F   짙은 코발트. 헤더, 이정표 판
--ink-2:   #1D3B5E
--granite: #E7E8E2   갈리시아 화강암. 페이지 배경
--stone:   #C6C8BE   테두리, 구분선
--flecha:  #F0B429   노란 화살표
--vino:    #7A2E39   도장 잉크
--moss:    #3F5D4A
--text:    #1B2430
--muted:   #6B7280
```

**노란색(`--flecha`) 사용 규칙: 길 안내·현재 위치·활성 상태에만.**
실제 카미노에서 노란색은 방향 지시 전용이다. 장식이나 강조에 남용하면 규칙이 무너진다.

**서체**
- 표제: `Gowun Batang` (한글 세리프) — 아껴서
- 본문: `IBM Plex Sans KR`
- 수치: `IBM Plex Mono` — 거리·고도·남은 km는 전부 모노 + `tabular-nums`

**시그니처: 이정표(모호온) 판.** 상단에 산티아고까지 남은 거리를 스페인식 소수점 표기로 (`114,235`). 이 컴포넌트는 임의로 바꾸지 않는다.

### 갈림길과 위험

- 프랑스 길은 **11곳에서 갈라진다.** 같은 순례길이라도 사람마다 걷는 길이 다르다
- **나폴레옹 루트 11/1~3/31 폐쇄.** 겨울에는 발카를로스가 유일한 선택지
- 위험 유형 8종: 급경사 하강(무릎) · 대피소 없음 · 그늘 없음 · 식수 없음 · 차도 병행 · 진흙 · 통신 두절 · 겨울 결빙
- **위험 표시는 겁주기가 아니라 준비하기 위한 것이다.** 대응 행동을 반드시 함께 적는다

### 콤포스텔라 요건 (2025년 개정) — 자주 틀린다

| 방식 | 최소 거리 |
|---|---|
| 도보·말 | **100km** |
| 자전거 | **200km** |
| **전기자전거** | **대상 아님** |

**조건**
- 연속이어야 한다. **이동수단으로 건너뛰면 그 지점에서 끊긴다**
- 스페인 영토 안에서 채워야 한다
- **마지막 구간이 최소 20km이고 대성당으로 직접 이어져야 한다**
- 국외 출발이면 스페인 내 최소 70km

**⚠️ 도장 규칙 — 위치가 아니라 총 도보 거리 기준**

| 조건 | 하루 도장 |
|---|---|
| 최소 거리만 걷는 경우 (도보 100km대) | **2개** |
| 그보다 길게 걷는 경우 | **1개** |

**생장에서 773km를 걸으면 사리아를 지나도 하루 1개면 된다.**
❌ `if (지점 > 사리아) 도장 2개` — 틀린 로직이다

걷지 않은 구간은 **안개 지도 개방과 도보 거리 집계에서 제외**한다.

### 이동 방식과 접근 교통

**⚠️ 순례자는 걷기만 하지 않는다. 이 프로젝트를 도보 전용으로 만들지 마라.**

| 방식 | 콤포스텔라 | 하루 이동 |
|---|---|---|
| 도보 | 100km | **3~32km** — 하한을 두지 마라 |
| 자전거 | 200km | 40~100km |
| 전기자전거 | **대상 아님** | — |
| 전동 핸드바이크 | 확인 필요 | 평균 20km |
| 휠체어·조엘레트 | 확인 필요 | 3~15km |
| 말 | 100km | — |

- `TravelMode`는 위 6종. **Phase 1~3은 도보만 지원하되 모델은 처음부터 갖는다**
- **하루 목표 거리 하한을 14km로 잡지 마라.** 의족을 쓰고 하루 3~4km 걷는 계획이 실재한다
- 방식이 바뀌면 **위험 부위도 바뀐다.** 도보는 발·무릎, 핸드바이크는 손목·어깨
- 순례는 걷기 전부터 시작한다. **인천 → 생장은 최소 2~3회 환승**이고 준비에서 가장 헷갈리는 부분이다
- 생장에는 공항이 없다. 파리(CDG TGV) · 비아리츠 · 팜플로나 경유가 주요 경로
- 바욘 → 생장 기차는 하루 6편, 약 12유로, 1시간
- **여정 중에도 버스·택시를 쓴다.** 부상·만실·시간 부족 때문이다. 이걸 예외로 보지 않는다

### ⚠️ 이동수단은 Plan B가 아니다

**부르고스 → 레온 약 200km의 메세타를 버스로 건너뛰는 것은 흔한 선택이다.**
포럼에서 끊임없이 논의되고 상당수가 실제로 그렇게 한다.

- `PlannedTransport`로 **일정 수립 단계에서** 넣을 수 있어야 한다
- ❌ 이동수단 사용을 "계획 실패"로 처리하는 UI·문구
- ✅ 걷지 않은 구간은 조용히 제외하고, 콤포스텔라 요건에 걸릴 때만 경고
- **어느 쪽도 권하지 않는다.** 시간과 체력의 현실을 보여주고 판단은 사용자가 한다

### 완주 이후 (Phase 3~5)

- **콤포스텔라** — 도보 100km 이상 완주 시 대성당이 발급하는 공식 증서
- **「내 돌」** — 집에서 가져온 돌을 철의 십자가(1,505m)에 놓는 의식. **마스코트를 만들지 않고 이미 있는 것을 기록한다**
- **안개 지도** — 걸으면 열린다. 랜드마크 19곳. ⚠️ **경로·화살표·알베르게는 절대 가리지 않는다**
- **이중 순례자(Dual Pilgrim)** — 카미노와 일본 구마노고도를 모두 걸은 사람에게 주는 공식 자격. 세계에서 단 둘뿐인 유네스코 세계유산 순례길이다. 2015년 시작, 2025년 5월 1만 번째 수여
  - 받는 것: 조가비 + 세 발 까마귀(야타가라스) 핀 배지, 화지 수제 인증서
  - **카미노를 먼저 걸으면** 구마노 혼구 대사에서 다이코 의식에 참여할 수 있다. 반대 순서면 없다
  - 등록: 구마노 혼구 헤리티지 센터 또는 타나베시 관광안내소
  - **우리 확장 전략(구마노고도)의 실제 진입점이다**

### 실제 애로사항 (제품 설계의 근거)

1. **침대 확보 불안이 1번 문제다.** 길은 노란 화살표 덕에 거의 안 잃는다. 순례자는 오늘 그 마을에 침대가 남을지 몰라서 새벽 5시에 뛴다. 대부분은 남는다. **정보 부재의 문제다**
2. **부상이 완주 실패의 주된 원인.** 그래서 일정 목표는 최단 일수가 아니라 **부상 없는 완주 확률**이다
3. **언어 장벽이 1번을 악화시킨다.** 전화 한 통이면 사립 예약이 되는데 스페인어를 못 해서 못 건다
4. **빈대(chinches).** 발생 시 여정이 망가진다. 해법은 **고온 건조기**이므로 `hasDryer`가 중요하다

## 작업 방식

1. 작업 전 관련 파일을 먼저 읽는다. 추측으로 수정하지 않는다
2. 한 번에 하나의 관심사만 변경한다
3. 계산 로직을 건드리면 `npm test`를 돌리고 결과를 보고한다
4. 새 의존성을 추가하기 전에 이유를 먼저 설명한다
5. 커밋 메시지는 한국어, 명령형 (`일정 분할 엔진에 오르막 회복 보정 추가`)
6. **확실하지 않으면 지어내지 말고 질문한다**
