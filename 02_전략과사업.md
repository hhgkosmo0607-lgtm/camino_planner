# 1부 · 사업모델

### 1. 목표

> **한국에서 카미노를 준비하거나 걷는 모든 사람이 이 서비스를 쓴다.**

#### 1.1 이 목표는 타당한가

**타당하다. 단, 보통의 이유와는 반대 방향에서 타당하다.**

일반적으로 100% 점유는 비현실적이다. 시장이 크면 경쟁자가 반드시 생기고, 취향이 갈리고, 어떤 세그먼트는 절대 안 넘어온다.

**그런데 이 시장은 연 1만 명이다.** 너무 작아서:

- 글로벌 경쟁자(Buen Camino, Wise Pilgrim)가 한국어에 투자할 이유가 없다 — 한국인은 전체 순례자의 **1.4%**다
- 국내 대기업이 진입할 이유가 없다 — 최대 매출이 수억 원대다
- 반대로 **한 팀이 전부를 감당할 수 있는 크기다** — 연 1만 명이면 CS도 데이터 관리도 소수 인원으로 가능하다

> **작은 시장은 약점이 아니라 해자(moat)다. 아무도 안 오는 시장이라서 전부 가질 수 있다.**

#### 1.2 "100%"의 정의

정의를 잘못 잡으면 측정도 달성도 못 한다.

**잘못된 정의**

| 정의 | 왜 틀렸나 |
|---|---|
| 앱 설치 100% | 불가능. 패키지 이용자, 60대 일부는 앱을 안 깐다 |
| 유료 전환 100% | 불가능하고, 시도하면 도달률이 무너진다 |
| 완주자 100% | 가장 도움이 필요한 중도 포기자를 놓친다 |

**채택하는 정의**

> **도달률(Reach) = 그 해 카미노를 준비하거나 걸은 한국인 중, 이 서비스를 한 번이라도 사용한 사람의 비율**

- "사용"의 최소 단위는 **웹 일정 계산기 1회 실행.** 설치도 가입도 불필요
- 분모 추정 가능: 콤포스텔라 한국인 수령자 + 중도 이탈 추정치
- **실제로 측정 가능한 지표다.** 순례자 사무소가 매년 국적별 통계를 공표한다. 대부분의 스타트업은 자기 점유율을 모르는데, 우리는 안다

#### 1.3 단계 목표

| 연도 | 도달률 | 절대 사용자 | 근거 |
|---|---|---|---|
| 2027 (성년) | 30% | 약 4,000 | SEO + 실측 콘텐츠로 검색 유입 확보 |
| 2028 | 60% | 약 6,500 | 완주자 입소문 + 카페·협회 침투 |
| 2029 | 85% | 약 9,000 | 사실상 표준 |
| 이후 | 85~90% 유지 | — | 100%는 점근선 |

**85%를 "전부"로 간주한다.** 마지막 15%(패키지 이용자, 비디지털층)를 잡는 비용이 앞의 85%보다 크다.

### 2. 이 목표가 강제하는 제약

목표를 도달률로 잡는 순간, 아래는 선택이 아니라 **전제조건**이 된다.

#### 제약 1 — 핵심 기능은 영원히 무료

유료화는 도달률의 상한을 만든다. Wise Pilgrim이 $4.99를 받는 순간 사용자는 "돈 낼 의향이 있는 사람"으로 한정된다.

- 일정 계산, 마을 정보, 숙소 정보, 오프라인 지도 — **전부 무료**
- 수익은 **사용자가 어차피 쓸 돈**(장비·보험·항공·짐배송)에서 나온다

> **이전 계획의 프리미엄 구독 ₩19,900은 폐기한다.** 순례기 PDF·포토북 같은 사후 상품이면 도달률을 해치지 않으므로 그 형태로만 남긴다.

#### 제약 2 — 가입을 요구하지 않는다

회원가입은 도달률을 30~50% 깎는다.

- 계획 상태는 **URL 쿼리스트링에 담는다** → 링크 공유가 곧 바이럴
- 계정은 순례 일지처럼 저장이 필요한 기능에서만 선택적으로

#### 제약 3 — 웹이 본체, 앱은 부속

앱 설치는 큰 마찰이다. 특히 60대에게.

```
검색 → 웹에서 즉시 일정 계산 (설치·가입 없음)
     → 링크 공유
     → 출국 2주 전, 오프라인 지도가 필요할 때만 앱 설치
```

**앱을 안 깐 사람도 우리 사용자로 카운트한다.** 이게 도달률 정의의 핵심이다.

#### 제약 4 — 모든 세그먼트를 동시에

타깃을 좁힐 수 없다. 20대 최저예산 배낭족과 68세 은퇴자를 **같은 제품으로** 만족시켜야 한다.

다행히 요구사항이 겹친다.

| | 20대 | 30대 | 60대 |
|---|---|---|---|
| 침대 확보 불안 | ✓ | ✓ | ✓ |
| 부상 회피 | ✓ | ✓ | ✓✓ |
| 언어 장벽 | ✓ | ✓ | ✓✓ |
| 비용 최적화 | ✓✓ | ✓ | — |
| 큰 글씨·단순 UI | — | — | ✓✓ |

**공통 문제가 압도적으로 크다.** 세그먼트 분기는 글꼴 크기와 비용 강조 정도면 충분하다.

### 3. 관문 분석 — 어디를 잡아야 하는가

한국인 순례자가 **반드시 지나가는 지점**을 찾아 거기에 자리 잡는다.

| 관문 | 통과 비율 | 장악 가능성 | 비고 |
|---|---|---|---|
| **검색 (네이버·구글)** | **~100%** | **높음** | 유일한 완전 관문 |
| 항공권 예약 | 100% | 낮음 | 개입 여지 적음 |
| 장비 구매 | ~95% | 중간 | 제휴 가능하나 시점이 늦음 |
| 네이버 카페 | 70~80% | 중간 | 직접 광고 불가, 기여로만 접근 |
| 크레덴시알 발급 | 100% | **낮음** | 대부분 생장 현지 발급. 국내 단일 창구 없음 |
| 여행사 | 20~30% | 중간 | B2B 제휴 |
| 본당·수도회 | 10% | 중간 | 단체 순례 |

#### 결론: 검색이 유일한 100% 관문이다

**그러므로 SEO는 마케팅 채널이 아니라 전략 그 자체다.**

이전 계획에서 SEO를 "획득 채널의 40%"로 잡았던 것과 다르다. 도달률이 목표라면 **자원의 대부분을 여기 쏟아야 한다.**

### 4. 실행 설계

#### 4.1 검색 장악 — 롱테일 총력전

한국인이 카미노 준비 중 검색하는 모든 질문에 우리 페이지가 있어야 한다.

| 유형 | 예시 | 페이지 수 | 생성 방식 |
|---|---|---|---|
| 마을 | "포르토마린 알베르게", "오 세브레이로 숙소" | 82 | **데이터 자동 생성** |
| 구간 | "생장에서 론세스바예스", "사리아 포르토마린 거리" | 81 | **데이터 자동 생성** |
| 일정 | "산티아고 순례길 20일 코스", "사리아 5일" | 40+ | 반자동 |
| 준비 | "산티아고 배낭 무게", "순례자 여권 발급" | 60+ | 수기 |
| 상황 | "알베르게 만실이면", "빈대 물렸을 때", "무릎 아플 때" | 50+ | 수기 |
| 비용 | "산티아고 순례길 비용 30일" | 20+ | 반자동 |

**총 330페이지 이상. 이 중 163개는 이미 확보한 82개 마을 데이터에서 자동 생성된다.**

> 콘텐츠 농장이 아니다. 각 페이지는 실제로 유용해야 하고, 끝에 일정 계산기로 연결된다.

#### 4.2 무료 도구 — 각각이 검색 유입구

가입도 설치도 없이 즉시 쓰는 도구를 독립 URL로 배포한다.

| 도구 | 유입 기대 | 비고 |
|---|---|---|
| 일정 계산기 | ★★★ | 핵심. 링크 공유 가능 |
| 비용 계산기 | ★★★ | 구간·기간·숙소 유형별 |
| 배낭 무게 계산기 | ★★★ | 체중 10% 기준 |
| 준비 타임라인 생성기 | ★★☆ | 출국일 입력 → D-90 체크리스트 |
| 스페인어 문장 생성기 | ★★☆ | 전화 예약·약국·빈대 신고 |
| 짐배송 비용 비교 | ★★☆ | 공립 vs 사립 트레이드오프 |

**각 도구가 독립적으로 유입을 만들고 서로를 연결한다.**

#### 4.3 완주자를 채널로

카미노에서는 매일 밤 한국인끼리 정보를 교환한다. 이 구조를 이용한다.

- 순례 종료 시 **순례기 PDF 자동 생성** → SNS·블로그 게시 시 자연 노출
- 도장 수집 화면을 공유하기 좋게 설계
- **"이거 뭐 써요?" 하고 물었을 때 답이 되는 것**이 목표

#### 4.4 비디지털층 — 마지막 15%

60대 이상, 여행사 패키지 이용자에게는 앱이 안 닿는다.

- **인쇄용 일정표 PDF** — A4 한 장. **자녀가 뽑아서 부모에게 준다**
- 본당 순례 설명회용 자료 무료 배포
- 여행사에 일정표 생성 도구 제공 → **역으로 여행사가 우리를 쓰게 만든다**

> **"자녀가 부모를 위해 대신 계획을 세우는" 경로가 실제로 존재한다.** 사용자와 수혜자가 다른 케이스를 설계에 반영한다.

#### 4.5 마케팅 채널 비중

| 채널 | 방법 | 비중 |
|---|---|---|
| 검색 (SEO) | 330페이지 + 무료 도구 | **60%** |
| 유튜브 | 실측 도보 40일 연재 | 20% |
| 네이버 카페 | 정보 기여 후 자연 언급 (**직접 광고 금지**) | 10% |
| 천주교 본당 | 순례 준비 설명회 | 5% |
| 기타 | 등산·아웃도어 커뮤니티 | 5% |

> 네이버 카페는 노골적 홍보에 매우 적대적이다. 무료 계산기를 공개하고 질문에 성실히 답하는 방식으로만 접근한다.

### 5. 사업 모델 — 부상과 숙소, 두 축

#### 5.1 왜 이 두 가지인가

01 문서의 애로사항 순위 1·2위가 이것이다.

1. **침대 확보 불안** — 하루를 시계 중심으로 돌아가게 만든다
2. **부상** — 완주 실패의 주된 원인

그런데 **03 문서 2부의 여행사 상품 분석이 이걸 그대로 증명한다.**

거의 모든 패키지가 **"전 일정 짐 이동(동키 서비스)"** 를 첫 줄에 내세우고, 알베르게에 호텔을 섞어 **숙소 확보의 확실성**을 판다. 585~720만 원짜리 상품이 파는 것의 본질이 **짐(=부상)과 숙소**다.

**시장이 이미 이 두 가지에 200만 원의 값을 매기고 있다.**

| | 자유여행 | 패키지 | 차액 |
|---|---|---|---|
| 총비용 | 370~490만 | 585~720만 | **약 200~250만** |

차액의 내용은 짐 배송 + 숙소 확보 + 인솔이다.
**우리는 앞의 두 개를 50만 원 이하로 해결한다. 이게 사업의 전부다.**

#### 5.2 두 축은 하나의 사슬로 연결된다 ⭐

이게 이 사업 모델의 핵심 통찰이다.

```
부상을 피하려면  →  짐을 부쳐야 하고
짐을 부치려면    →  숙소가 미리 확정돼야 하고
                    (공립은 짐 배송을 안 받는다)
숙소를 확정하면  →  사립 예약이 필요하고
사립을 예약하면  →  우리에게 수수료가 들어온다
```

**억지 판매가 아니다.** 사용자에게 실제로 이득이고, 그 과정에서 자연스럽게 수익이 발생한다.

반대 방향도 성립한다.

```
숙소가 불안하면  →  새벽에 뛰게 되고
새벽에 뛰면      →  무리하게 되고
무리하면         →  다친다
```

**숙소 불안이 부상을 만든다. 두 문제는 원래 하나였다.**

#### 5.3 축 1 — 부상

> **부상은 400만 원짜리 여정을 망치는 유일한 요인이다. 그래서 지불 의사가 가장 높다.**

##### 개입 시점과 수익화

| 시점 | 개입 | 수익 | 근거 |
|---|---|---|---|
| **D-90** | 부상 회피 일정 설계 | — (무료, 유입) | 초반 과부하가 부상의 주원인 |
| **D-60** | **신발·배낭·폴 안내 → 제휴** | **R-A1** | 잘못된 신발이 물집의 1번 원인. **길들이기 기간 때문에 D-60에 사야 한다** |
| D-30 | 12주 훈련 계획 | — (무료, 리드) | |
| D-30 | 상비약 목록 | R-A2 | 소염진통제·근육이완제가 실제로 부족했다는 후기 다수 |
| **D-14** | **여행자 보험 → 제휴** | **R-A3** | 패키지도 1억 보험을 포함한다 |
| **여정 중** | **짐 배송 판단·접수** | **R-A4** | 부상 위험 감소의 직접 수단 |
| 여정 중 | 통증 대응 안내 | — | ⚠️ 전문가 검수 필수 |
| 여정 중 | 물리치료사·약국 위치 | — | 경로상 물리치료사가 실재한다 |
| 여정 중 | 부상 시 일정 재조정 + 대체 교통 | — | 버스·택시 구간 안내 |

##### ⭐ 개입 타이밍이 특정된다

관찰된 패턴이 있다.

- 첫 주가 가장 힘들고, 몸은 **4~7일**이면 적응한다
- 그런데 **프랑스 길에서 부상 순례자가 가장 많은 시점은 2주차**다

즉 **몸이 적응했다고 느끼는 순간 무리해서 다친다.**

**→ 8~14일차에 개입한다.**

```
"지금쯤 몸이 적응했다고 느끼실 겁니다.
 이 구간에서 무리하다 다치는 경우가 많습니다.
 오늘 거리를 조금 줄이시겠어요?"
```

⚠️ "통계적으로"라는 표현은 근거를 확보한 뒤에만 쓴다. 현재는 커뮤니티 관찰 수준이다.

##### 이 축의 방어선

경쟁 앱이 따라하기 어렵다. **일정 엔진 + 고도 데이터 + 한국인 체력 기준 + 장비 제휴**가 결합돼야 한다.

> **우리는 예약을 팔 수 없다. 대신 "언제 예약해야 하는지"를 판다.**

**그게 정확히 짐 배송을 쓰는 사람이 필요로 하는 것이다.**

##### ⭐ 첫날과 마지막날이 확실한 수익 지점이다

**생장피드포르 첫날 숙소**는 사실상 전원이 예약한다. 도착이 늦고 다음 날 피레네를 넘어야 한다.
**산티아고 도착 후 숙소**도 마찬가지로 대부분 2~3일 머문다.

**이 두 지점만으로도 예약 중개가 성립한다.** 여정 중 예약은 그 위에 얹는다.

##### 예약을 직접 운영하지 않는다

자체 예약 시스템은 만들지 않는다(→ 03 문서 1부 W1). 대신:

- **예약 플랫폼 어필리에이트** — 사립 알베르게 상당수가 등록돼 있다
- **전화·WhatsApp 원터치 + 스페인어 스크립트** — 플랫폼에 없는 곳
- 우리는 **판단과 연결**만 한다

⚠️ **잘못된 "예약 가능" 표시는 사람을 한데서 자게 한다.** 미확인은 `UNKNOWN`으로 둔다.

### 6. 100%가 사주는 것과 못 사주는 것

#### 못 사주는 것 — 매출

```
도달률 100% × 12,000명 × ARPU ₩30,000 = 연 3.6억
```

**점유율 100%를 달성해도 연 매출 3~4억이다.** 2~3인 팀의 생계는 되지만 투자 유치 스토리는 아니다. **이건 반드시 직시해야 한다.**

#### 사주는 것 1 — 협상력의 역전

- 점유율 30%: 우리가 여행사에 "송객해드릴게요" 하고 부탁한다
- 점유율 85%: **여행사가 우리에게 "노출시켜주세요" 하고 온다**

같은 일인데 수수료율이 다르다. 장비 브랜드, 보험사도 마찬가지다. 위 재무 추정에서 송객 단가가 25만 → 32만으로 오르는 근거가 이것이다.

#### 사주는 것 2 — 데이터 독점

한국인 순례자가 언제 출발하고, 어디서 시작하고, 며칠에 걷고, **어디서 포기하는지** — 이 데이터를 가진 곳이 없다. 콤포스텔라 통계는 완주자 결과만 보여준다.

여행사·관광청·장비사에 팔 수 있고, 무엇보다 **제품을 계속 정확하게 만든다.**

#### 사주는 것 3 — 다음 시장으로의 이전 (가장 중요)

**카미노 100%는 목적지가 아니라 교두보다.**

"한국인 순례자 전원이 쓰는 서비스"라는 사실은 다음 시장 진입 시 그대로 신뢰 자산이 된다.

| 다음 시장 | 규모 | 이전되는 것 |
|---|---|---|
| 제주올레 (437km) | 연 90만+ | 구간 분할 엔진, 브랜드 신뢰 |
| 해파랑길 (750km)·남파랑길 | 수십만 | 동일 |
| 일본 시코쿠 88 순례길 | 성장세 | 순례 + 도장 문화가 동일 |
| **구마노고도** | 카미노 자매길 | **완주자가 다음으로 가는 길** |

특히 구마노고도는 **세계에서 단 둘뿐인 유네스코 세계유산 순례길**로서 카미노와 짝을 이루며, 양쪽을 걸은 사람에게 **이중 순례자(Dual Pilgrim)** 자격을 주는 공식 제도가 2015년부터 운영되고 있다. 2025년 5월에 1만 번째 수여자가 나왔다.

**이 제도가 확장 전략의 실제 형태다.**

- 카미노 완주자에게 "아직 절반입니다"라고 말할 근거가 있다
- 일본이라 항공권 20~30만 원, 3~5일이면 된다
- 카미노를 먼저 걸은 사람은 구마노에서 **다이코 의식**에 참여할 수 있다. 순서가 유리하다
- **한국 순례자 커뮤니티에서 거의 다뤄지지 않는다**

즉 **완주 보상이자 다음 시장으로의 다리**를 하나의 기능이 동시에 수행한다. 상세 설계는 03 문서 1부 6.4절.

> **"카미노 도달률 100% → 도보여행 카테고리"** 가 실제 사업 경로다.
> 카미노만 보면 3.6억이지만, 카테고리를 잡으면 자릿수가 달라진다.

### 8. 권고 — 어떻게 시작할 것인가

#### 8.1 하지 말아야 할 것

> **이 아이템으로 처음부터 2억을 태워 2인 전업 창업하지 않는다.**

위 재무 추정에서 1년차 8,300만 원 적자는 성년 효과를 최대로 잡은 낙관 시나리오다. 2028년 반동까지 감안하면 2년간 1억 원 가까이 태우게 된다.

#### 8.2 권고 경로

**Phase 0 — 사이드 프로젝트 (2026 하반기, 약 900만 원)**

- **인터뷰 20명 → 트래픽 4주 → 실측 도보 40일** 순서로 검증한다 (→ 05 문서)
- 실측 도보는 휴가나 안식월로 처리
- **최악의 경우에도 카미노를 한 번 걷고 포트폴리오가 남는다. 실패 비용이 낮다**

|---|---|
| 인터뷰 사례 20명 | 60만 |

**Phase 1 — 성년 시즌 실측 (2027 상반기)**
- 트래픽, 이메일 수집, 제휴 문의를 측정
- 판단 기준 (→ 05 문서 7장 게이트 4와 일치시킨다):
  - 도달률 **20% 이상**
  - 제휴 매출 전망 **연 5,000만 이상**
  - 앱 현장 사용률 **40% 이상**
  - **셋 중 둘 이상 충족 → 전업 전환.** 그 미만이면 사이드 유지 또는 정리

**Phase 2 — 전업 및 확장 (2027 하반기~)**
- 정부지원 신청, 2인 체제

- 2028년부터 제주올레·구마노고도 확장 검토

#### 8.3 왜 이 순서인가

목표가 도달률 100%라면 **초기에 돈을 쓸 곳이 별로 없다.** 필요한 건 검색 페이지와 계산기이고, 둘 다 자본이 아니라 시간과 데이터의 문제다. 진짜 비용은 **40일 실측 도보**인데 이건 어차피 해야 하는 일이다.

# 2부 · 패키지 수요 전환

### 0. 왜 이 시장을 노리는가

**패키지 고객 1명 = 자유여행 고객 10~15명의 가치.**

| 구분 | 우리 매출 |
|---|---|
| 자유여행자 (도달 사용자) | ARPU 2~3만 |
| **패키지 전환자** | **20~50만** |

3년차 기준으로 계산하면 이렇다.

```
패키지 검토층 (연 1.2만 명 중 20~30%)   = 2,400~3,600명
그중 "불안해서" 사는 사람 (약 60%)        = 1,400~2,200명
우리가 15% 전환                          = 210~330명
객단가 30만                              = 6,300만~9,900만
```

**자유여행 전체 도달(8,900명)에서 나오는 매출과 맞먹는다.**

---

### 1. ⭐ 그들은 왜 사는가

#### 1.1 오해 — "돈이 많아서 산다"

**틀렸다. 불안해서 산다.**

비용 차이를 실제로 계산하면 생각보다 작다.

| 항목 | 자유여행 (35일) | 패키지 |
|---|---|---|
| 항공 | 150만 | 포함 |
| 숙박 | 알베르게 35박 × 15유로 ≈ 52만 | 포함 (호텔·펜시온 위주) |
| 식비 | 35일 × 30유로 ≈ 147만 | 일부 포함 |
| 장비 | 60~120만 | 별도 |
| 짐 배송 | 0 (안 씀) | 포함 (약 200유로 상당) |
| 기타 | 20만 | |
| **합계** | **약 430~490만** | **약 500~600만** |

**차액은 70~150만 원에 불과하다.**

> **그런데 왜 대부분은 자유여행을 갈까?**
> **돈이 아니라 "알베르게에서 자고 싶어서"다.**

#### 1.2 패키지가 실제로 파는 것

| 파는 것 | 우리 축 | 우리가 줄 수 있나 |
|---|---|---|
| 숙소 확정 | B | **가능** (필요한 날만) |
| 짐 배송 | A+B | **가능** (필요한 구간만) |
| 준비 부담 제거 | 전부 | **가능** |
| 언어 지원 | A+B | **가능** (보여주기 카드) |
| 문제 발생 시 대응 | A+B | **부분 가능** |
| **가족 안심** | — | **가능. 아무도 안 함** |
| 인솔자 동행 | — | **불가** |
| 동행자 (외롭지 않음) | — | **불가** |

**8개 중 6개를 우리가 준다. 그것도 훨씬 싸게.**

#### 1.3 패키지의 치명적 약점

> **⭐ 패키지로 가면 알베르게에 안 묵는다.**

호텔과 펜시온에 묵는다. 그런데 **순례 경험의 대부분이 알베르게에서 일어난다.**

- 매일 밤 세계 각국 순례자와 섞이는 것
- 공동 식사, 공동 빨래, 공동 소등
- 도장을 받고 침대를 배정받는 의식
- "부엔 카미노" 인사가 오가는 공간

**패키지 고객은 100만 원을 더 내고 순례를 잃는다.**

⚠️ 다만 이걸 **공격적으로 말하지 않는다.** "여행사는 나쁘다"가 아니라 **"이런 선택지도 있습니다"** 로 제시한다.

---

### 2. 세그먼트 — 누구를 뺏을 수 있나

**패키지 검토층은 하나가 아니다. 뺏을 수 있는 사람과 없는 사람을 구분한다.**

| # | 유형 | 비중(추정) | 뺏을 수 있나 | 이유 |
|---|---|---|---|---|
| **P1** | **불안해서** | 30% | **✅ 높음** | 안심 장치를 주면 된다 |
| **P2** | **알아볼 시간이 없어서** | 20% | **✅ 높음** | 준비를 대신해주면 된다 |
| **P3** | **가족이 걱정해서** | 15% | **✅ 높음** | **가족 안심 장치. 아무도 안 함** |
| P4 | 언어가 두려워서 | 15% | 🟡 중간 | 보여주기 카드로 상당 부분 |
| P5 | 혼자가 외로워서 | 10% | ❌ 낮음 | 동행은 못 만들어준다 |
| P6 | 진짜 편하게 가고 싶어서 | 10% | ❌ 낮음 | **뺏을 필요도 없다** |

**→ P1+P2+P3 = 65%가 전환 가능 시장이다.**

#### 2.1 ⭐ P3 — 가족이 걱정해서 (가장 놓치기 쉬운 세그먼트)

**60대 부모를 보내는 자녀가 결제자인 경우가 많다.**

| 자녀의 걱정 | 패키지가 주는 답 | 우리가 줄 수 있는 답 |
|---|---|---|
| 무슨 일 생기면? | 인솔자가 있다 | **매일 도착 알림 + 비상 체계** |
| 길을 잃으면? | 가이드가 있다 | 오프라인 지도 + 카드 |
| 아프면? | 인솔자가 병원에 데려간다 | **의료 정보 카드 + 보험 + 카드 A4** |
| 말이 안 통하면? | 통역이 있다 | **보여주기 카드 23장** |
| 연락이 안 되면? | 인솔자에게 연락 | **3일 미도착 시 자동 확인** |

> **"가족 안심"은 여행사도 제대로 안 판다. 그리고 우리 원가는 거의 0이다.**

### 3. 상품 설계 — 두 단계

#### 3.1 왜 두 단계인가

**여행업 규제 때문이다.** (→ 02 문서 4부 3장)

숙소 예약과 짐 배송을 **대행**하면 여행 상품 판매다. 관광사업자 등록이 필요하다.
그래서 **규제를 안 건드리는 상품**과 **화이트라벨 상품**을 나눈다.

#### 3.2 셀프형 — "안심 준비" (15~25만) · 규제 없음

**우리는 판단과 도구만 준다. 실행은 본인이 한다.**

| 구성 | 내용 |
|---|---|
| ① 맞춤 일정 | 체력·기간·나이 반영. 부상 회피 설계 |
| ② **예약할 날 지정** | "이 5일은 잡으세요. 나머지는 비워두세요" + 숙소 후보와 **원터치 예약 링크** |
| ③ **플랜 B 목록** | 각 도착지의 대안 숙소 3곳 + 연락처 |
| ④ 짐 배송 판단 | 어느 구간에 쓸지 + 접수 방법 안내 |
| ⑤ 보험 체크리스트 | 무엇을 봐야 하는지 (도보 담보 포함 여부 등) |
| ⑥ 장비 안내 | 신발 중심. **안 사도 되는 것은 안 사도 된다고 말한다** |
| ⑦ **보여주기 카드** | 앱 + 인쇄본 (A4 양면 + 명함형 의료 카드) |
| ⑧ **가족 안심 세트** | 아래 3.4절 |
| ⑨ 출국 전 점검 | D-7 화상 30분 (선택) |

**핵심**: 우리는 예약을 **대신 하지 않는다.** 링크를 만들어주고 사용자가 누른다.

#### 3.3 대행형 — "안심 동행" (40~60만) · 화이트라벨

**셀프형 전부 + 실제 실행 대행.**

| 추가 구성 | 실행 주체 |
|---|---|
| 숙소 예약 대행 (5~8일) | **제휴 여행사** |
| 짐 배송 사전 접수 | **제휴 여행사** |
| 첫날·마지막날 숙소 | **제휴 여행사** |
| **한국어 비상 지원** | 우리 (3.5절) |

```
우리          : 일정 설계, 숙소 선정, 판단, 도구, 가족 안심
제휴 여행사    : 실제 예약, 결제, 계약 주체, 사고 대응
```

⚠️ **이 구조의 적법성을 반드시 확인한다.** (H18)

#### 3.4 ⭐ 가족 안심 세트 — 킬러 기능

**원가 거의 0, 경쟁자 없음, P3 세그먼트를 통째로 가져온다.**

| 기능 | 내용 |
|---|---|
| **매일 도착 알림** | 순례자가 도착 표시 → 가족에게 자동 문자/카톡 |
| **미도착 알림** | 예정 시각 + 4시간 경과 시 가족에게 알림 |
| **3일 규칙** | 3일간 기록이 없으면 가족에게 확인 요청 |
| **오늘 어디쯤** | 가족용 웹 페이지. 앱 설치 불필요 |
| **비상 연락 카드** | 대사관, 보험사, 현지 응급번호 |
| **의료 정보 카드** | 03 문서 3부 A7 |
| 완주 알림 | 산티아고 도착 시 축하 메시지 |

⚠️ **위치 실시간 공유는 하지 않는다.** (프라이버시, 배터리)
**"도착했다"는 사실만** 전달한다. 순례자가 직접 누른다.

> **자녀가 부모에게 사주는 상품이 될 수 있다.**
> 결제자와 사용자가 다른 구조를 설계에 반영한다.

#### 3.5 한국어 비상 지원 (대행형만)

**시차가 오히려 유리하다.** 스페인 낮 = 한국 저녁.

| 시간 | 상황 |
|---|---|
| 스페인 14시 (한국 22시) | 알베르게 도착 시간대 |
| 스페인 18시 (한국 새벽 2시) | 저녁 |

**한국 시간 저녁~밤에 카톡으로 대응한다.** "오늘 만실인데 어떡하죠" 같은 문의.

⚠️ **사람이 필요해서 확장이 어렵다.** 대행형 100명까지가 한계. 그 이상은 자동화하거나 가격을 올린다.
⚠️ **의료 문의에는 답하지 않는다.** (→ 02 문서 4부 2.5)

### 4. 어떻게 만나는가 — 전환 깔때기

#### 4.1 3단계 장벽

```
① 인지    "패키지 말고 다른 방법이 있다"를 모른다
              ↓
② 신뢰    "정말 괜찮을까"
              ↓
③ 실행    "그래도 뭘 해야 할지 모르겠다"
```

#### 4.2 ① 인지 — 검색어를 가로챈다

**패키지를 검토하는 사람이 검색하는 말은 자유여행자와 다르다.**

| 검색어 | 현재 검색 결과 | 우리 대응 |
|---|---|---|
| 산티아고 순례길 패키지 | 여행사 랜딩페이지뿐 | **정직한 비교 페이지** |
| 산티아고 순례길 여행사 추천 | 광고성 글 | 비교 + 우리 대안 |
| 산티아고 순례길 비용 | 파편적 | **비용 계산기** |
| 산티아고 혼자 갈 수 있나 | 후기 산발 | **셀프 진단** |
| **60대 산티아고 순례길** | 거의 없음 | **공백. 반드시 잡는다** |
| **부모님 산티아고 순례길** | 거의 없음 | **P3 진입점** |
| 산티아고 순례길 준비 얼마나 걸려 | | 타임라인 |
| 산티아고 패키지 vs 자유여행 | **거의 없음** | **핵심 페이지** |

> **"60대", "부모님" 검색어에 제대로 된 한국어 콘텐츠가 거의 없다. 여기가 가장 큰 공백이다.**

#### 4.3 ⭐ 핵심 콘텐츠 — "정직한 비교"

**여행사를 깎아내리지 않고, 숫자로 보여준다.**

```
산티아고 순례길, 얼마나 들까 — 세 가지 방법

                   자유여행    안심 준비    패키지
─────────────────────────────────────────────────
항공                150만       150만      포함
숙박              알베르게52만  알베르게62만  호텔 포함
식비                147만       147만      일부 포함
장비                 80만        80만       80만
짐 배송                 0     오르막 4만       포함
서비스                  0        20만          —
─────────────────────────────────────────────────
합계                429만       463만    500~600만

어디서 자나        알베르게    알베르게      호텔
일정 변경          자유        자유        불가
준비 부담          전부 본인   대신 해줌   없음
문제 생기면        스스로      한국어 지원  인솔자
```

**그리고 정직하게 덧붙인다.**

> **이런 분은 패키지가 맞습니다.**
> · 혼자가 아니라 사람들과 함께 가고 싶다
> · 준비를 아예 하고 싶지 않다
> · 알베르게 다인실이 부담스럽다
>
> **[제휴 여행사 상품 보기]**

⚠️ **패키지가 맞는 사람은 여행사로 보낸다.** 이게 신뢰를 만들고, 동시에 R6 송객 수수료가 된다.

#### 4.4 ② 신뢰 — 먼저 준다

| 순서 | 행동 |
|---|---|
| 1 | 무료 계산기로 일정을 만들어본다 |
| 2 | 결과가 구체적이고 정확하다 (부상 경고, 침대 수, 하강) |
| 3 | 실측 도보 콘텐츠로 "진짜 걸어봤구나" |
| 4 | **"안 사도 됩니다" 를 말하는 걸 본다** |
| 5 | 그제서야 상품을 본다 |

**무료 도구가 신뢰를 만들고, 신뢰가 전환을 만든다.**

#### 4.5 ③ 실행 — 진단으로 밀어준다

**"저는 어떤 방법이 맞을까요?" 진단**

```
Q1. 나이대는?
Q2. 며칠 걸으실 건가요?

→ 결과: 자유여행 / 안심 준비 / 안심 동행 / 패키지 권장
```

⚠️ **패키지가 나오면 그대로 권한다.** 우리 상품으로 억지로 끌지 않는다.
이 정직함이 나머지 전환을 만든다.

### 5. 여행사와의 관계 — 충돌하는가

#### 5.1 문제

**우리가 패키지 고객을 뺏으면 여행사가 적이 된다.** 그런데 R5(화이트라벨)와 R6(송객)은 여행사가 필요하다.

#### 5.2 해소 — 파이를 나누는 게 아니라 넓힌다

```
현재
  자유여행 (80%) ─────────── 패키지 (20%)
       ↑                        ↑
   여행사 매출 0            여행사 매출

우리가 만드는 것
  자유여행 (65%) ── 반패키지 (15%) ── 패키지 (20%)
                        ↑
                  여행사가 실행 → 여행사 매출 발생
```

**반패키지는 여행사에게 새로운 상품 라인이다.**

- 우리가 기획하고 여행사가 실행 → **여행사에 없던 매출**
- 자유여행자였던 사람이 15~25만 원짜리 실행 서비스를 산다
- 여행사는 마케팅·기획 비용 없이 고객을 받는다

#### 5.3 여행사에 이렇게 제안한다

```
"저희 고객은 지금 여행사를 이용하지 않는 분들입니다.

패키지가 부담스러워 혼자 준비하다가,
결국 숙소 예약이나 짐 배송에서 막히는 분들이 많습니다.

저희가 일정을 설계하면, 실행만 맡아주실 수 있을까요.
기존 패키지보다 단가는 낮지만 물량이 있고,
마케팅 비용이 들지 않습니다.

그리고 저희 진단에서 '패키지가 맞다'고 나오는 분들은
그대로 여행사 상품으로 안내드립니다."
```

**마지막 문장이 핵심이다.** 우리는 여행사의 경쟁자가 아니라 **깔때기 위쪽**이다.

### 2. 🔴 의료 정보 — A축의 근본 위험

#### 2.1 위험의 실체

우리는 **부상 예방과 대응**을 사업의 한 축으로 삼는다. 그런데:

| 행위 | 위험 |
|---|---|
| "이 통증은 쉬면 낫습니다" | **진단·의료 판단** |
| "이 약을 드세요" | **처방 행위** |
| "무릎보호대를 착용하세요" | 의료기기 관련 표현 |
| `injuryRiskScore` 78점 | **의학적 근거 없는 수치 제시** |
| 부상 가이드 45개 페이지 | 의료 정보 제공 |

**의료 자문 없이 이걸 하면 안 된다.**

#### 2.2 우리가 지킬 선

```
증상을 전달한다              ← 우리
        ↓
전문가가 판단한다             ← 약사·의사
        ↓
판단을 정확히 받아온다         ← 우리
        ↓
일정에 반영한다               ← 우리
```

**우리는 통로다. 판단자가 아니다.**

#### 2.3 필수 조치

| # | 조치 | 시점 |
|---|---|---|
| 1 | **정형외과 또는 스포츠의학 자문 계약** | 부상 콘텐츠 착수 전 |
| 2 | 부상 가이드 45개 전량 검수 | 게시 전 |
| 3 | 보여주기 카드 의료 8장 검수 | 배포 전 |
| 4 | **`injuryRiskScore`를 숫자로 노출 금지** | 즉시 |
| 5 | 약 이름·용량·복용법 미기재 | 즉시 |
| 6 | 전 페이지 면책 고지 | 게시 전 |
| 7 | 응급 시 112 안내 상시 노출 | 즉시 |

#### 2.4 면책 고지 문안 (초안 — 변호사 검토 필요)

### 3. 🔴 여행업 규제 — R5 반패키지

#### 3.1 문제

**숙소 예약 + 짐 배송 + 보험을 묶어 팔면 여행 상품이다.**

국내에서 여행 상품을 기획·판매하려면 **관광사업자 등록**과 **영업보증보험**이 필요하다.

#### 3.2 세 경로

| 경로 | 규제 | 판단 |
|---|---|---|
| A. 순수 제휴 (링크만) | 없음 | 안전하나 수익 얇음 |
| B. 직접 판매 | **등록 필요** | 부담 큼 |
| **C. 화이트라벨** | **경계가 모호** | **택하되 반드시 확인** |

#### 3.3 확인해야 할 것

```
□ "기획만 하고 판매는 등록 여행사가 한다"가 법적으로 성립하는가
□ 우리가 받는 것이 "기획료"인가 "알선 수수료"인가 — 성격에 따라 규제가 다를 수 있다
□ 사고 발생 시 책임 소재 (여행사 / 우리 / 공급자)
□ 계약 주체가 누구로 표시되는가
□ 우리 화면에서 결제가 일어나면 판매로 보는가
```

⚠️ **H18로 05 문서에 등록했다. 법률 자문 없이 R5를 진행하지 않는다.**

### 4. 🔴 보험 제휴 — 모집 행위 규제

#### 4.1 문제

여행자 보험을 소개하고 수수료를 받으면 **보험 모집 행위**에 해당할 수 있다.

국내에서 보험을 모집하려면 **보험설계사·대리점 등록**이 필요하다.

#### 4.2 가능한 구조

| 구조 | 설명 | 규제 |
|---|---|---|
| 단순 정보 제공 | 상품 비교표만 제공, 링크 없음 | 낮음 |

#### 4.3 확인해야 할 것


⚠️ **R1이 매출의 3분의 1이다. 여기가 막히면 사업모델을 다시 짜야 한다.**
**H16(제휴 가능성)과 함께 법적 가능성도 동시에 확인한다.**

### 7. 🟠 커머스와 광고

#### 7.1 제휴 표시 의무

**대가를 받고 상품을 추천하면 표시해야 한다.**

```
□ "제휴 링크를 통해 수수료를 받습니다" 명시

```

#### 7.2 ⚠️ 공포 마케팅 금지 — 자체 원칙

법적 문제이기 이전에 **윤리와 신뢰의 문제다.**

| 금지 | 이유 |
|---|---|
| "다칠 수 있으니 사세요" | 공포 마케팅 |
| 부상 통계로 상품 유도 | 위와 동일 |
| 의료 정보 페이지에 상품 링크 | **정보와 상품의 분리** |

**원칙: 부상 정보는 무료로, 상품 연결 없이 제공한다.**
장비 추천은 **일정 설계의 결과로** 나온다.

#### 7.3 의료기기·건강기능식품

무릎보호대, 깔창, 테이핑 등은 **의료기기로 분류될 수 있다.**

```
□ 취급 품목의 의료기기 해당 여부
□ 의료기기 광고 사전심의 대상 여부
□ 효능·효과 표현 금지 범위
```

### 8. 🟡 서비스 책임

#### 8.1 "판단을 제공한다"의 무게

우리 차별점은 "데이터가 아니라 판단"이다. **판단에는 책임이 따른다.**

| 상황 | 위험 |
|---|---|
| "오늘은 서두르지 않아도 됩니다" → 만실 → 노숙 | 손해배상 주장 |
| "이 일정은 무리 없습니다" → 부상 | 손해배상 주장 |
| 숙소 정보 오류 → 헛걸음 | 불만·환불 요구 |
| 개방 기간 오류 → 문 닫힌 알베르게 | 위와 동일 |

#### 8.2 대응

| # | 조치 |
|---|---|
| 1 | **모든 판단에 근거를 함께 표시** ("침대 240개, 6월 평일 기준") |
| 2 | **"추정"임을 항상 명시** |
| 3 | `verifiedAt` 노출로 데이터 나이를 밝힌다 |
| 4 | 미확인 정보는 `UNKNOWN`으로 표시하고 추정하지 않는다 |
| 5 | 이용약관에 면책 조항 |
| 6 | 무료 서비스임을 명확히 (유료 부분과 구분) |

> **정직함이 최선의 방어다.** "우리도 모른다"고 쓰는 것이 잘못된 정보보다 안전하다.
