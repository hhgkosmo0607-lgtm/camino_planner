## 1. 시스템 개요

### 1.1 아키텍처

```

│  · 오프라인 우선      │   │  · 준비 단계 중심     │
│  · SQLite 로컬 DB    │   │  · SEO 유입 창구      │
└──────────┬───────────┘   └──────────┬───────────┘
           │                          │
           └────────────┬─────────────┘
                        │ REST / JSON
              ┌─────────▼──────────┐
              │  API 서버           │
              │  Node.js + Postgres │
              │  · 데이터 배포       │
              │  · 사용자 동기화     │
              │  · 제보 수집         │
              └─────────┬──────────┘
                        │
        ┌───────────────┼───────────────┐
   지도 타일        데이터 소스        결제
   (MapTiler/       (협회·Gronze·    (토스/Stripe)
    OSM 자체호스팅)   현지 실측)
```

### 1.2 기술 선택 근거

| 항목 | 선택 | 근거 |
|---|---|---|
| 앱 | React Native | 웹 프로토타입 코드 재사용, 2인 팀 규모에 적합 |
| 로컬 DB | SQLite (WatermelonDB) | 오프라인 우선. 전체 마을 DB 20MB 이내 |
| 지도 | OSM + 자체 타일 호스팅 | Google Maps는 오프라인 캐싱 불가 |
| 경로 | GPX + 단말 내 매칭 | 서버 통신 없이 이탈 감지 |
| 백엔드 | Node.js + PostgreSQL(PostGIS) | 지리 쿼리, 인력 확보 용이 |
| 동기화 | 최종 수정 우선(LWW) | 순례자는 단일 기기 사용, 충돌 드묾 |

### 1.3 비기능 요구사항

| 항목 | 요구 수준 |
|---|---|
| **오프라인** | 항공기 모드에서 지도·경로·숙소·일정 전 기능 동작. 온라인 필요 기능은 회색 처리 |
| **배터리** | GPS 상시 추적 시 8시간 걷기에 배터리 25% 이내 소모 (10초 간격 샘플링) |
| **저장 공간** | 프랑스 길 전 구간 지도 포함 400MB 이내 |
| **접근성** | 최소 글꼴 17pt, 대비 4.5:1 이상, 60대 사용자 대상 테스트 필수 |
| **응답 속도** | 일정 재계산 300ms 이내 (로컬 연산) |
| **언어** | 한국어 단일. 단, 알베르게·마을명은 원어 병기 필수 (현지에서 보여줘야 함) |

## 3. 기능별 상세 사양

### F-01 구간 자동 계획

**입력**
| 항목 | 형식 | 기본값 |
|---|---|---|
| 루트 | 프랑스 길 / 레온 출발 / 사리아 출발 | 프랑스 길 |
| 출발일 | 날짜 | — |
| 가용 일수 **또는** 하루 목표 거리 | 정수 / 14~32km | 24km |
| 체력 수준 | 낮음 / 보통 / 높음 | 보통 |
| 휴식일 | 0~5일 | 0 |

**처리 로직**

```
1. 목표 거리 산정
   - 일수 입력 시:  daily = 총거리 / (일수 - 휴식일)
   - 거리 입력 시:  daily = 입력값
   - 체력 보정:     낮음 ×0.85 / 보통 ×1.0 / 높음 ×1.15

2. 구간 분할 (탐욕 + 후처리)
   현재 위치에서 목표거리 지점을 계산하고,
   침대가 있는 마을 중 가장 가까운 곳을 도착지로 선택.
   산티아고 도달까지 반복.

3. 후처리 보정
   a. 마지막 구간 < 6km  → 직전 구간에 병합
   b. 오르막 400m 초과   → 목표거리 15% 하향 후 재계산
      (피레네, 오 세브레이로, 폰세바돈 3개 구간)
   c. 연속 3일 28km 초과 → 경고 표시
   d. 침대 30개 미만 마을 → "성수기 만실 주의" 배지

4. 검증
   - 총 거리 100km 미만 시 콤포스텔라 미발급 경고
   - 사리아 이후 구간에 "도장 2개 필요" 안내 삽입
```

**출력**: 일자별 카드 (도착지, 거리, 오르막, 예상 소요, 침대 수, 도착 예정일)

**엣지 케이스**
- 하루 목표 14km 미만 → 숙소 없는 구간 발생, 하한선 고정
- 전 구간 + 목표 32km → 25일. 물리적으로 가능하나 부상 경고 노출
- 출발지가 산티아고 100km 이내 → 콤포스텔라 요건 미달 사전 안내

### F-02 오프라인 지도 및 경로 안내

**사전 다운로드**
- 계획 확정 시 해당 루트 구간의 지도 타일(줌 12~16) + GPX 자동 다운로드
- Wi-Fi 전용 다운로드 기본값, 진행률 표시, 중단 후 재개 지원
- 프랑스 길 전 구간 기준 약 320MB

**주행 중 동작 (전량 오프라인)**
| 기능 | 동작 |
|---|---|
| 현재 위치 | GPS + GPX 매칭, 남은 거리 표시 |
| 경로 이탈 | 트랙에서 **150m 이상** 벗어나면 진동 + 알림 |
| 다음 서비스 | "식수 2.1km / 바 4.8km / 오늘 도착지 11.3km" 상시 표시 |
| 갈림길 | 대안 경로 분기점 사전 알림 (예: 사모스 vs 산실 우회) |
| 배터리 절약 | 화면 꺼짐 시 60초 간격 샘플링 전환 |

> **이탈 임계값 150m 근거**: 실제 카미노는 마을 통과 시 GPX와 실제 도보 경로가 50~100m 차이 나는 구간이 흔하다. 임계값이 낮으면 알림 피로가 발생한다.

### F-03 숙소 정보

**목록 정렬**: 계획된 도착 마을 순 → 마을 내에서는 공립 → 기부제 → 사립

**카드 표시 항목**
- 원어명 (크게, 현지에서 보여줄 수 있게)
- 유형 배지 / 침대 수 / 요금 / 예약 가능 여부
- **"○년 ○월 확인" 검증일**
- 한국인 메모 (예: "취사 가능, 근처 슈퍼 20시 마감")

**혼잡 예측** (v1.1)
```
혼잡도 = f(요일, 계절, 마을 총 침대 수, 사용자 제보)
표시:  여유 / 서두르세요 / 만실 가능성 높음
```
예약 기능은 제공하지 않는다. 대부분의 공립 알베르게가 온라인 예약 인프라를 갖고 있지 않으며, 잘못된 예약 가능 표시는 순례자를 노숙 위험에 빠뜨린다. 대신 **전화·WhatsApp 원터치 연결 + 스페인어 예약 문장 자동 생성**을 제공한다.

### F-04 순례 일지 및 도장

- 도착 표시 → 자동으로 거리·소요시간 기록, 도장 생성
- 도장은 마을명 기반 SVG 자동 생성 + 실물 도장 사진 첨부 가능
- 사리아 이후 구간에서는 "오늘 도장 2개" 리마인더 (콤포스텔라 요건)
- 오프라인 작성 → 온라인 복귀 시 자동 동기화
- 여정 종료 시 PDF 순례기 내보내기 (v1.1)

**주의**: 산티아고 대성당이 QR 기반 공식 디지털 크레덴시알을 운영 중이다. 우리 도장은 **공식 증빙이 아닌 개인 기록**임을 명시해야 하며, 콤포스텔라 발급에는 실물 크레덴시알이 필요함을 앱 내에서 반복 안내한다.

## 4. 데이터 확보 계획

이 서비스의 실질적 진입장벽은 코드가 아니라 데이터다.

| 단계 | 방법 | 비용 | 비고 |
|---|---|---|---|
| 1차 | 공개 데이터 정리 (OSM, 갈리시아 주정부 공개 GPX) | 0 | 라이선스 확인 필수 |
| 2차 | 대한민국 산티아고 순례자 협회 제휴 | 협의 | 국내 유일 크레덴시알 발급처, 숙소 DB 보유 |
| 3차 | **현지 실측 도보 (2026년 가을, 40일)** | 약 800만 | 창업자 직접 완주. 전 마을 검증 + 콘텐츠 확보 |
| 4차 | 사용자 제보 (연중) | 0 | 제보 3건 채택 시 프리미엄 1개월 지급 |

> **3차 현지 실측은 선택이 아니라 필수다.** 카미노를 걸어보지 않은 팀이 만든 앱은 순례자 커뮤니티에서 즉시 판별되며, 신뢰를 얻지 못한다. 동시에 이 40일이 마케팅 콘텐츠(연재·유튜브)의 원천이 된다.

## 6. 미해결 사항

1. **지도 타일 라이선스** — OSM 자체 호스팅 vs MapTiler 유료. 오프라인 대량 캐싱 시 약관 확인 필요
2. **의료 정보 책임 범위** — 물집·통증 가이드에 대한 법적 검토 및 의료 자문 계약 필요
3. **협회 제휴 조건** — 데이터 제공 대가, 독점 여부
4. **iOS 백그라운드 위치 권한** — App Store 심사 시 상시 위치 사용 사유 소명 준비
5. **결제 수단** — 국내 결제만 할지, 해외 거주 한국인 대상 카드 결제 지원할지
