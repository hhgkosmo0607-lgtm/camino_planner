## 0. 이 서비스가 무엇인지

한국인 순례자가 **다치지 않고(A축), 숙소 걱정 없이(B축)** 걷게 하는 서비스다.

| 축 | 문제 | 기능 |
|---|---|---|
| **A. 몸** | 부상이 완주를 막는다 | 부상 회피 일정(F-01) · 장비 안내(F-13) · 보험 체크(F-14) · 짐배송 판단(F-05) |
| **B. 잠자리** | 어느 날 예약할지 모른다 | 혼잡 추정(F-02) · 숙소 정보(F-03) · 예약 스크립트(F-04) · Plan B(F-21) |

**제약 셋: 무료 · 가입 없음 · 오프라인 동작.** 이 셋이 모든 설계 판단의 기준이다.

### 0.1 데이터 모델은 `schema.ts`가 정본이다

이 문서에 나오는 TS 인터페이스는 **설명을 위한 발췌**다. 실제 구현은 별도 파일 `schema.ts`를 따른다.

| 항목 | 내용 |
|---|---|
| 파일 | `schema.ts` (별도 첨부) |
| 검증 | `tsc --strict --noEmit` 통과 확인됨 |
| 범위 | 타입 31개 — Town·SegmentProfile·RouteFork·Albergue·PlanInput·Stage·Plan·MobilityProfile·PlannedTransport 등 |

**개발 착수 시 이 파일을 그대로 `lib/schema.ts`로 복사하고, 이후 모든 타입은 여기서 import 한다.** 문서에 적힌 타입을 베껴 쓰면 드리프트가 생긴다.

### 0.2 Phase 구분을 반드시 확인한다

**Phase 1에서 실제로 만드는 것은 F-01(구간 자동 계획)뿐이다.** 나머지 기능은 전부 Phase 2 이후이며, 일부는 현지 실측 도보로 데이터를 먼저 확보해야 시작할 수 있다.

| **1** | F-01 구간 자동 계획, 마을·구간 SEO 페이지, 무료 도구 3종, 인쇄용 일정표 |

### 2.2 SegmentProfile — 필수

```ts
interface SegmentProfile {
  fromTownId: string
  toTownId: string
  distanceKm: number
  ascent: number       // 누적 상승 m
  descent: number      // 누적 하강 m
  maxElevation: number
  source: 'OSM+MDT' | 'ESTIMATED'
}
```

#### 왜 별도 테이블인가

**마을 고도 차이로 오르막을 계산하면 크게 틀린다.**

| 생장 → 론세스바예스 | |
|---|---|
| `도착지 − 출발지` | +780m |
| 실제 (레포에데르 고개 약 1,450m 경유) | **약 +1,280m** |

카미노에서 부상이 가장 많은 날인데 "평범한 산길"로 판단된다.

**하강 누락도 같은 급의 문제다.** 무릎 부상의 주원인은 내리막이다. 엘 아세보 → 몰리나세카는 1,145m → 595m로 −550m 급경사 돌길인데, 상승 기준으로는 0이라 "쉬운 날"이 된다.

**→ `ascent`/`descent`는 반드시 실제 경로 좌표와 수치표고모델에서 계산한다.**

| 데이터 | 출처 | 라이선스 |
|---|---|---|
| 경로 | OpenStreetMap 하이킹 릴레이션 | ODbL |
| 고도 | 스페인 IGN MDT05 (LiDAR 5m 격자) | CC BY 4.0 |

⚠️ **고도를 그대로 더하면 노이즈 때문에 실제의 2~3배가 나온다.** 이동평균과 임계값(4m)으로 평활화한다.
검증: **메세타 평야 구간(부르고스 → 오르니요스)의 상승이 100m 이하여야 정상이다.**

⚠️ `source`가 `ESTIMATED`인 동안은 위험 점수를 사용자에게 숫자로 노출하지 않는다.

**주요 갈림길**

|---|---|---|
| **생장피드포르** | **나폴레옹(고지)** vs **발카를로스(계곡)** | **11/1~3/31 나폴레옹 폐쇄** |
| 오바노스 직전 | 직행 vs 에우나테(12세기 성당) | 역사 |
| 부르고스 진입 | N-120 vs 강변길 | 차도 회피 |
| 사아군 이후 | 엘 부르고 라네로 vs 칼사다 로마나 | 한적함 |
| **레온 이후** | 비야당고스 vs **비야르 데 마사리페** | 차도 회피 |
| 비야프랑카 이후 | 직행 vs 프라델라(능선) | 경치 vs 급경사 |
| **트리아카스텔라** | 산 실 vs **사모스(대수도원)** | 거리 vs 볼거리 |
| 그 외 4곳 | | |

⚠️ **나폴레옹 루트는 11월 1일~3월 31일 폐쇄된다.** 오리송 위로 대피소가 없고 날씨가 몇 분 만에 바뀐다. 당일 안전 판단은 생장 순례자 사무소가 하며, **우리는 사전 정보만 제공한다.**

### 2.3 Albergue

```ts
type AlbergueType =
  | 'MUNICIPAL' | 'XUNTA' | 'PARISH'
  | 'MONASTERY' | 'PRIVATE' | 'DONATIVO'

type ReservationMethod =
  | 'NONE'      // 선착순 — 예약 불가
  | 'PHONE' | 'WHATSAPP' | 'ONLINE'
  | 'UNKNOWN'   // 미확인. 추정으로 채우지 않는다

interface Albergue {
  id: string
  townId: string
  name: string              // 원어명
  type: AlbergueType
  beds: number | null
  priceEur: number | null   // 0 = 기부제, null = 미확인
  reservation: ReservationMethod
  contact: string | null
  openFrom: string | null   // MM-DD. 겨울 폐쇄 다수
  openTo: string | null
  hasKitchen: boolean | null
  hasLaundry: boolean | null
  hasDryer: boolean | null            // 빈대 대응의 핵심
  hasHeating: boolean | null
  acceptsBagTransfer: boolean | null  // 공립은 대부분 false
  verifiedAt: string | null           // YYYY-MM. UI에 노출
  source: 'FIELD' | 'PARTNER' | 'USER_REPORT' | 'PLACEHOLDER'
}
```

#### 어기면 안 되는 것

**① `reservation`을 추정으로 채우지 않는다.**
공립은 예약 불가가 일반적이지만 하드코딩하지 않는다. 2025년에 성수기 예약 허용 제안이 있었고 정책은 바뀔 수 있다. 미확인은 `UNKNOWN`으로 두고 UI에서 "확인 필요"로 표시한다.

**② `verifiedAt`을 UI에 노출한다.**
"2026년 4월 확인" 표기는 오래된 데이터에 대한 클레임을 구조적으로 방어한다.

**③ `openFrom`/`openTo` 없이 숙소를 추천하지 않는다.**
공립은 보통 부활절~10월 말 운영이다. 11월 순례자에게 4월에만 여는 곳을 추천하면 그 사람은 한데서 잔다.

**④ `acceptsBagTransfer`는 마을 단위와 별개다.**
마을에 `BAG_TRANSFER`가 있어도 **공립 알베르게는 짐 배송을 받지 않는다.** 예약을 안 받아 침대를 보장할 수 없기 때문이다.

## 3. 기능

### F-01 구간 자동 계획

**목표 함수는 최단 일수가 아니라 부상 없는 완주 확률이다.**

#### 입력

| 항목 | 값 | 기본 |
|---|---|---|
| 루트 | 프랑스 길 / 레온 / 사리아 | 프랑스 길 |
| 기간 또는 하루 목표 거리 | 정수 / 14~32km | 24km |
| 체력 | 낮음 / 보통 / 높음 | 보통 |
| 휴식일 | 0~5 | 전 구간 1, 100km 0 |
| 짐 배송 | 안 함 / 오르막만 / 전 구간 | 안 함 |

#### 알고리즘

```
1. 목표 거리
   기간 입력 시:  daily = 총거리 / (일수 - 휴식일)
   체력 보정:     낮음 ×0.85 / 보통 ×1.0 / 높음 ×1.15

2. 탐욕 분할
   현재 위치 + 목표거리 지점에 가장 가까운 beds > 0 마을을 도착지로.
   ※ 짐 배송 사용 시 acceptsBagTransfer 숙소가 있는 마을만 후보

3. 부상 회피 후처리  ← 핵심
   a. 초반 적응: 1~3일차 ×0.8
   b. 오르막 보정: 다음 구간 상승 600m 초과 시 ×0.85
   c. 회복 배치: 상승 600m 또는 하강 500m 초과 구간의 다음 날 ×0.85
   d. 휴식일: beds >= 150 인 큰 도시에 배치
   e. 마지막 구간 < 6km → 직전 구간에 병합

4. 경고 산출

5. injuryRiskScore
   평균거리, 최대거리, 누적 상승, 누적 하강(가중), 초반부하, 연속고강도
   ※ 하강 가중치를 상승보다 높게 (무릎 부상의 주원인)

6. 검증
   총 거리 < 100km → 콤포스텔라 미발급 경고
   총 도보 거리가 100km대면 → 하루 도장 2개 안내
```

#### 출력

일자별 카드 — 도착지(한글+원어), 거리, 상승·하강, 예상 소요, 침대 수, 경고 배지, 도착 예정일.

상단에 한 줄 판단: *"무리 없는 일정입니다"* / *"초반 3일이 다소 빡빡합니다. 하루 늘리는 것을 권합니다"*

### F-02 "오늘 서두를 필요 있나요?"

**기존 도구가 답하지 않는 질문이다.**

순례자는 오늘 그 마을에 침대가 몇 개고 몇 명이 향하는지 알 방법이 없다. 그래서 최악을 가정하고 새벽에 뛴다.

#### 추정 모델

```
예상 수요 = 기준 순례자 수
          × 계절 계수 × 요일 계수
          × 구간 계수 (사리아 이후 급증)
          × 특수일 계수

혼잡도 = 예상 수요 / 마을 총 침대 수
```

**특수일**: 부활절, 7월 25일(성 야고보), 10월 12일(스페인 국경일), 7월 6~14일(팜플로나 산 페르민), **2027년 전체(성년)**

#### 출력

```
팔라스 데 레이 · 침대 240개
⚠ 6월 토요일 · 사리아 출발자 집중 구간
```

### F-12 보여주기 카드

**화면을 상대에게 내밀면 대화 없이 상황이 끝난다.**

#### 번역기와 다른 점

| | 번역기 | 카드 |
|---|---|---|
| 전제 | 내가 뭘 말할지 안다 | **뭘 말해야 하는지 우리가 안다** |
| 상대의 답 | 못 알아듣는다 | **선택지를 짚으면 한국어로** |
| 네트워크 | 필요 | **오프라인** |
| 맥락 | 없음 | 이름·인원·일정 자동 삽입 |

#### 카드 구조

```
┌────────────────────────────────┐
│  🐚  Peregrino coreano          │
├────────────────────────────────┤
│   상대가 읽을 문장 (22~26pt)     │
│   [그림 또는 도식]               │
├────────────────────────────────┤
│  ↓ Señale su respuesta          │
│  [ Sí ] [ No ] [ ... ]          │  ← 상대가 누른다
├────────────────────────────────┤
│  (내가 읽는 작은 한국어 안내)     │
└────────────────────────────────┘
```

#### 목록 (23장)

| 분류 | 장수 | 주요 카드 |
|---|---|---|
| 숙소 | 9 | 침대 있나요 · **만실인데 어디로** · 짐배송 · 건조기 · 빈대 |
| 몸 | 8 | 약국(물집·근육통·관절) · 진료 · **응급** · **내 의료 정보** · 보험 서류 |
| 이동·기타 | 6 | 택시 · 버스 · 길을 잃음 · 물 · 식사 제한 |

#### 대표 흐름 — 약국

약사가 "Descanse ___ días"에서 **3**을 누르면:

```
"3일 쉬세요"

일정을 3일 미루면
도착이 11월 8일 → 11월 11일

[일정 다시 짜기]  [숙소 연장 문의 카드]
```

**약사의 판단이 곧바로 일정 재계산으로 이어진다.** 번역기가 못 하는 지점이다.

#### 만실 대응 카드

```
Está completo. ¿A dónde puedo ir?

[ Ventas de Narón   4.5 km  ~1h ]
[ Palas de Rei     12.4 km  ~3h ]
[ Gonzar (atrás)    3.0 km ~45m ]
```

마을 목록과 거리는 우리 데이터에서 자동 생성된다.

#### 안전 원칙

```
증상을 정확히 전달한다      ← 우리
        ↓
전문가가 판단한다           ← 약사·의사
        ↓
판단을 정확히 전달받는다     ← 우리
        ↓
일정에 반영한다             ← 우리
```

**우리는 통로다. 판단자가 아니다.**

| 금지 |
|---|
| 약 이름·용량·복용법을 우리가 쓰지 않는다 |
| 진단하지 않는다 |
| "괜찮다"고 안심시키지 않는다 |

의료 카드 하단 고정: *이 카드는 의료 조언이 아닙니다. 응급 시 112.*

#### 요건

| 항목 | 내용 |
|---|---|
| 저장 | 전량 로컬. 오프라인 동작 |
| 표시 | 카드 표시 중 화면 밝기 최대 |
| 조작 | 한 손. 3초 안에 꺼낼 수 있어야 |
| 인쇄 | 핵심 8장 A4 양면 + 명함형 의료 카드 |
| **의료 정보 카드** | **단말 내 암호화. 서버 전송 금지.** 잠금화면 접근 옵션 |
| 문안 | 원어민 번역 + **현지 검수** (실측 도보에서 실제 사용) |

**생장피드포르에 공항이 없다.** 인천에서 최소 2~3회 환승.

| 경로 | 구간 | 특징 |
|---|---|---|
| 파리 경유 | 인천 → CDG → TGV 바욘 → 생장 | **CDG에 TGV역이 있다** |
| 비아리츠 경유 | 인천 → 파리 → 비아리츠 → 버스 → 바욘 → 생장 | 가장 가까운 공항 |
| **팜플로나 경유** | 인천 → 마드리드 → 팜플로나 → 생장 버스 | **가장 많이 쓴다** |

바욘 → 생장 SNCF 기차는 하루 6편, 1시간, 약 12유로.

일정의 **Day 0**으로 표시하고 귀환편까지 포함한다.

⚠️ **항공권을 우리가 예약하지 않는다.** 경로와 소요를 보여주고 예약은 사용자가 한다.
⚠️ 시각·요금은 자주 바뀐다. `verifiedAt` 표시 + 공식 사이트 링크.

### F-25 이동 방식과 접근성

| 방식 | 콤포스텔라 최소 | 하루 이동 | 주요 위험 부위 |
|---|---|---|---|
| 도보 | 100km | **3~32km** | 발·무릎 |
| 자전거 | **200km** | 40~100km | 안장통·손목·낙차 |
| **전기자전거** | **대상 아님** | — | — |
| 전동 핸드바이크 | 확인 필요 | 평균 20km | **손목·어깨** |
| 휠체어·조엘레트 | 확인 필요 | 3~15km | 어깨·욕창 |
| 말 | 100km | — | — |

```ts
interface MobilityProfile {
  mode: TravelMode
  maxKmPerDay: number          // ★ 하한 없음
  needsSupportVehicle: boolean
  needsCompanion: boolean
  avoidSurfaces: ('GRAVEL'|'MUD'|'STAIRS'|'STEEP'|'COBBLE')[]
  bagTransferRequired: boolean
}
```

⚠️ **하루 목표 거리 하한을 14km로 두지 않는다.** 하루 3~4km 계획이 실재한다.
⚠️ **방식이 바뀌면 위험 부위도 바뀐다.** 위험 판정을 방식별로 분기한다.
⚠️ 접근성 정보는 **틀리면 도보보다 훨씬 위험하다.** 확인된 것만 표시한다.
⚠️ **Phase 1~3은 도보 우선.** 다만 모델과 용어를 도보 전용으로 만들지 않는다.

### F-26 계획된 이동수단

**이동수단은 Plan B가 아니다.**

부르고스 → 레온 약 200km의 메세타를 버스로 건너뛰는 것은 흔한 선택이다.

```ts
interface PlannedTransport {
  fromTownId: string
  toTownId: string
  mode: 'BUS' | 'TRAIN' | 'TAXI' | 'SUPPORT_VEHICLE'
  reasonKo: string
}
```

일정 수립 단계에서 선택지로 제시한다.

| 항목 | 처리 |
|---|---|
| 도보 거리 | 제외 (`walkedKm`) |
| 콤포스텔라 | 연속 100km 판정에서 끊김 |
| 안개 지도 | 해당 구간 스팟 미개방 |
| 예산 | 교통비로 기록 |

⚠️ **어느 쪽도 권하지 않는다.** ❌ "계획 실패" 같은 문구 금지.
