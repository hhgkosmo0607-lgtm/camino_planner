# 카미노 프로젝트

> 한국인 순례자를 위한 카미노 데 산티아고 동반 서비스
> 문서 v6.4 · 2026년 7월

---

## 이 문서는

**프로젝트 전체 안내판이다.** 무엇을 만드는지, 문서가 어떻게 구성돼 있는지, 지금 어느 단계인지를 한 곳에서 본다.

> ⚠️ **산티아고 순례길을 모른다면 [부록 A · 순례길 입문](./A_순례길입문.md)을 먼저 읽는다.**
> 알베르게·크레덴시알·세요·콤포스텔라 같은 용어를 모르면 나머지 문서를 읽을 수 없다.

## 1. 한 장 요약

| 항목 | 내용 |
|---|---|
| **무엇** | 순례자가 다치지 않고, 숙소 판단을 대신 받으며 걷게 하는 서비스 |

| **최대 기회** | **패키지 수요 전환.** 고객 1명 = 자유여행 고객 10~15명 가치 |

## 2. 사업 모델

### 2.1 두 축

| 축 | 문제 | 파는 것 | 패키지가 대체하나 |
|---|---|---|---|
| **A. 몸** | 부상이 완주를 막는다 | 다치지 않게 하는 준비와 판단 | **아니다. 사도 다친다** |
| **B. 잠자리** | 어느 날 예약할지 모른다 | 예약과 자유 사이의 판단 | 그렇다 (비싸게) |

### 2.2 예약을 해도, 안 해도 부상으로 간다

```
  예약을 안 했다              예약을 다 했다
       ↓                          ↓
  자리가 걱정된다            오늘 25km를 가야만 한다
       ↓                          ↓
  새벽에 서둘러 걷는다        무릎이 아파도 걷는다
       ↓                          ↓
      부상  ←──────────────────→  부상
```

**판단이 없기 때문이다. 이 순환을 끊는 지점이 곧 우리가 돈을 버는 지점이다.**

### 2.3 최대 단일 기회 — 패키지 수요

| | 자유여행 | 패키지 |
|---|---|---|
| 총비용 | 약 430~490만 | 약 500~600만 |
| **차액** | | **70~150만** |
| 어디서 자나 | **알베르게** | **호텔** |

**그들은 돈이 많아서가 아니라 불안해서 산다. 그리고 그 100만 원에 알베르게를 잃는다.**

우리는 **20~50만 원에 안심을 주고 알베르게는 남긴다.**

⭐ **킬러 기능: 가족 안심 세트** — 매일 도착 알림, 3일 미도착 확인.
원가 0, 경쟁자 없음, **자녀가 부모에게 사주는 상품**이 된다.

### 2.4 역할 분담

| 축 | 역할 | 수단 |
|---|---|---|
| **B. 숙소** | **유입을 만든다** | 어느 날 예약할지 판단, 플랜 B 목록 |
| **A. 부상** | **수익을 만든다** | 보험, 신발·장비, 짐배송 |

순례자는 신발을 사러 오는 게 아니라 **"숙소 어떡하지"** 로 온다.

⚠️ **부상으로 물건을 팔지 않는다.** 신뢰가 이 사업의 유일한 자산이다.

### 2.5 서비스명 — 미확정

코드에서는 `config/brand.ts` 한 곳에서만 관리한다.

| 후보 | 뜻 | 약점 |
|---|---|---|
| **세요** (Sello) | 도장 | 뜻이 바로 안 통한다 |
| **모혼** (Mojón) | 이정표 | 한국인에게 생소 |
| **콘차** (Concha) | 조가비 | 흔하다 |
| **부엔** (Buen) | 좋은 | Buen Camino 앱과 충돌 |

확정 전 확인: 도메인 · 상표(여행·소프트웨어 분류) · 앱스토어 중복 · 스페인어권 부정적 의미 · 한국어 동음이의어 검색 경합

### 2.6 보상은 소유욕을 만들어야 한다

**PDF 한 장은 800km에 값하지 않는다.** 문제는 분량이 아니라 성격이다.

| 동인 | 우리 장치 |
|---|---|
| **노력의 증거** | **안개 지도 — 걸어야 열린다** |
| **수집 완성** | 세요(도장), 랜드마크 19곳 |
| **개인화** | 내 경로, 내 날짜, 내 사진 |
| **물성** | 포스터, 포토북, **일회용 필름 카메라** |
| **자랑 가능성** | 숫자 카드 |
| **애착** | **크레덴시알 · 「내 돌」** — 마스코트를 만들지 않고 이미 있는 것을 쓴다 |

**「내 돌」** — 순례자는 집에서 돌 하나를 가져와 **철의 십자가(1,505m)** 에 놓는다. 출발 전 사진과 한 줄을 등록해두면 그 지점에 도착했을 때 다시 보인다.

**안개 지도** — 지도가 구름에 덮여 있고 **걸으면 걸은 만큼 열린다.** 상징적 장소 19곳에 도착하면 주변이 밝아지며 이야기가 열린다. 완주 후 **"내가 밝힌 지도"가 포스터로 남는다.**

⚠️ **안개는 이야기 레이어만 가린다. 경로·화살표·알베르게는 항상 보인다.** 기본값은 꺼짐이고 언제든 끌 수 있다. **안전이 재미보다 위다.**

**⚠️ 90/10 원칙 — 위 표에서 무엇을 유료로 해도 되는가.** 비율이 아니라 경계선이다.
- "이걸 안 사면 성취가 덜해 보이나?" → YES면 무료 (노력의 증거·수집 완성·자랑 가능성·애착 — 표의 대부분)
- "이걸 사면 실물로 더 오래 간직·과시하나?" → YES면 유료 가능 (물성 행에 있는 포스터·포토북·필름 카메라 정도가 전부다)
- 억지로 비율을 맞추려고 무료 항목을 깎지 않는다

### 2.7 메뉴 명칭

**순례자가 현지에서 첫날부터 듣는 말만 쓴다.** 기독교 교리 용어는 쓰지 않는다.

| 메뉴 | 명칭 | 뜻 |
|---|---|---|
| 홈 | 오늘 | — |
| 일정 | 에타파 Etapa | 하루 구간 |
| 지도 | 플레차 Flecha | 노란 화살표 |
| 숙소 | 알베르게 Albergue | 순례자 숙소 |
| 기록 | 크레덴시알 Credencial | 순례자 여권 |
| 카드 | 카드 | — |
| 완주 | 콤포스텔라 Compostela | 완주 증서 |
| 준비 | 준비 | — |

한글을 크게, 원어를 작게 병기한다. **"카드"와 "준비"는 번역하지 않는다** — 다급하거나 낯선 사용자가 쓰는 곳이다.

## 4. 지금 어느 단계인가

### 4.1 Phase 구분 — 이것부터 확인한다

**웹과 앱은 순차(웹 완성 → 앱 착수)가 아니라 병렬 트랙이다.** 장기적으로는 앱이 더 중요한 채널이지만, 실측 도보·데이터 검증이 끝나지 않은 지금은 **웹 트랙만 실행 코드를 만든다.** 즉 "지금 만드는 게 웹뿐"인 이유는 우선순위가 아니라 **검증 순서** 때문이다 — 앱 트랙 착수를 특정 웹 지표(예: 과거의 "웹 도달률 20%") 달성에 종속시키지 않는다.

**지금 실제로 만드는 것은 F-01(구간 자동 계획)뿐이다.**

| Phase | 내용 | 조건 |
|---|---|---|
| **1** | 일정 계산기, 마을·구간 SEO 페이지, 무료 도구 3종, 인쇄용 일정표 | 지금 (웹 트랙) |
| 2 | 알베르게 데이터, 갈림길 추천, 일자별 상세, 예산, 접근 교통 | **실측 도보 완료 후** |
| 3 | 앱(React Native), 오프라인 지도, 보여주기 카드, 안개 지도, Plan B, 완주 화면 | **데이터·검증 완료 후 웹과 병행 착수** — 웹 트래픽 실적에 게이트하지 않는다 |
| 4~5 | 커머스, 접근성 지원, 루트 확장 | 제휴 확정 후 |

⚠️ **P0~P7 프롬프트에 Phase 2~3 기능이 섞이지 않게 한다.** 실제로 P0에 갈림길 데이터 생성이, P1에 갈림길 처리 로직이 끼어든 적이 있다(v5.7에서 수정).

> Phase 3 착수 조건의 구체적 지표(팀 규모, 착수 시점)는 아직 확정되지 않았다. 확정되는 대로 이 표와 06 문서 저장소 구조 결정을 함께 갱신한다.

### 4.2 🔴 개발 착수 전 반드시 해결할 것

**고도 데이터가 실제 오르막을 최대 39% 과소평가한다.**

| 생장 → 론세스바예스 (1일차) | |
|---|---|
| 마을 고도 차이로 계산 | +780m |
| 실제 (레포에데르 고개 약 1,450m 경유) | **약 +1,280m** |

그리고 **내리막을 전혀 고려하지 않는다.** 무릎 부상의 주원인은 내리막인데, 엘 아세보 → 몰리나세카(−550m 급경사 돌길)가 "쉬운 날"로 분류된다.

**해결 경로는 확보돼 있다.**

```
경로  ← OpenStreetMap 하이킹 릴레이션   (ODbL)
고도  ← 스페인 IGN MDT05               (CC BY 4.0, LiDAR 5m 격자)
      ↓
직접 계산 → ascent / descent
```

- **데이터 비용 0원.** IGN은 출처 표시만 하면 상업 이용 가능
- 작업 약 3일 (`scripts/build-profiles.ts`)
- ⚠️ **함정**: 고도를 그대로 더하면 노이즈 때문에 실제의 2~3배. 평활화 필수
- 개발 프롬프트 **P0.5**가 이 작업. **P1보다 먼저 한다**

### 4.3 검증되지 않은 것

이 문서들의 상당 부분은 **2차 자료 기반 가설**이다. 특히 아래는 **근거 없는 우리 창작**이다. 전체 목록은 01 문서 2부 2.6절.

- 페르소나 비율 (30/40/20/10)
- 도달률 목표 (30/60/85%)
- 전환율 전부 (보험 15%, 장비 12%, 숙소 10%…)
- 임계값 전부 (상승 600m, 하강 500m, 28km, 침대 30개)
- `injuryRiskScore` 공식 — **의학적 근거 전무**

## 5. 알아둘 제약 두 가지

### 5.1 프랑스 길은 하나의 선이 아니다

**같은 순례길이라도 사람마다 걷는 길이 다르다. 11곳에서 갈라졌다 다시 만난다.**

| 분기점 | 선택지 | 판단 요소 |
|---|---|---|
| **생장피드포르** | **나폴레옹(고지)** vs **발카를로스(계곡)** | **11/1~3/31 나폴레옹 폐쇄** |
| 레온 이후 | 비야당고스(차도) vs **비야르 데 마사리페** | 차도 회피 |
| 트리아카스텔라 | 산 실(짧음) vs **사모스(대수도원)** | 거리 vs 볼거리 |
| 그 외 8곳 | 에우나테 · 부르고스 강변길 · 칼사다 로마나 등 | |

**현재 데이터는 82개 마을의 단일 선형 배열이라 변형 경로가 없다.** 경로·고도 추출 시 변형도 함께 뽑는다 (03 문서 부록).

### 5.2 걷는 것만이 아니다

| 방식 | 빈도 | 콤포스텔라 |
|---|---|---|
| 도보 | 약 93% | 100km |
| **버스** | **흔하다 — 메세타 건너뛰기** | 해당 구간 제외 |
| 택시 | 흔하다 | 동일 |
| 자전거 | 약 4% | **200km** |
| 전동 핸드바이크 | 소수 | 확인 필요 |
| 휠체어·조엘레트 | 극소수 | 확인 필요 |
| 전기자전거 | 소수 | **대상 아님** |

**⚠️ 부르고스 → 레온 약 200km의 메세타를 버스로 건너뛰는 것은 흔한 선택이다.** 포럼에서 끊임없이 논의되고 상당수가 실제로 그렇게 한다. **우리 제품이 이걸 "계획 실패"로 취급하면 현실과 안 맞는다.**

**⚠️ 하루 목표 거리 하한을 14km로 두면 접근성 사용자가 배제된다.** 의족을 쓰고 하루 3~4km씩 걷는 계획이 실재한다.

**→ 제품의 정체성은 "걷기 계획기"가 아니라 "순례 여정 계획기"다.**

`TravelMode`는 처음부터 6종(`FOOT`·`BIKE`·`E_BIKE`·`HANDBIKE`·`WHEELCHAIR`·`HORSE`)을 갖는다. Phase 1~3은 도보를 우선하되, **나중에 넣으려면 전부 뜯어야 하기 때문이다.**
