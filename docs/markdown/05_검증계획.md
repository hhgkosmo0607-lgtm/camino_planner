## 0. 왜 이 문서가 필요한가

01~05 문서는 **무엇을 만들 것인가**로 채워져 있다. 그런데 그 근거의 상당수가 우리가 만든 숫자다.

01 문서 2부 2.6절의 등급 D·E 항목을 다시 보면:

- 페르소나 비율 30/40/20/10 — **창작**
- 도달률 목표 30/60/85% — **목표이지 예측이 아님**
- ARPU 3만 원, 수수료율 5~10% — **근거 없음**
- 임계값 전부 (상승 600m, 하강 500m, 28km, 침대 30개) — **임의 설정**
- `injuryRiskScore` 공식 — **의학적 근거 전무**
- 애로사항 ①~⑪ 전체 — **2차 자료 기반 가설**

**이 문서는 위 항목들을 사실로 바꾸는 계획이다.**

그리고 순서가 중요하다. **인터뷰를 먼저 하고, 그 결과로 실측 도보에서 무엇을 볼지 정한다.** 반대로 하면 40일을 걷고 와서 "그걸 안 봤네"가 된다.

**사례**: 3만 원 상당 (커피 기프티콘 또는 도서). 미리 명시한다.

### 3.3 ⚠️ 중도 포기자 인터뷰 주의

**이 사람들에게 순례 중단은 실패의 기억일 수 있다.**

- 모집 공고에 "실패", "포기" 같은 단어를 쓰지 않는다 → **"계획과 다르게 마무리하신 분"**
- 왜 못 걸었는지 캐묻지 않는다. 본인이 말하는 만큼만 듣는다
- 부상·건강 이야기가 나와도 진단하거나 조언하지 않는다
- 힘들어하면 **바로 중단한다.** 데이터보다 사람이 먼저다

### 3.4 진행 방식

- 1인 **45분** (패키지 관련 질문 해당자는 55분), 화상 또는 전화
- **녹음은 반드시 사전 동의.** 거절해도 진행한다
- 반구조화. 아래 질문지는 순서를 지킬 필요 없다
- **한 명이 진행하고 한 명이 기록**하면 좋지만, 혼자면 녹음에 의존

### 3.5 인터뷰 기법 — 이걸 지켜야 데이터가 쓸모 있다

| 원칙 | 나쁜 질문 | 좋은 질문 |
|---|---|---|
| **과거 행동을 묻는다** | "이런 앱 있으면 쓰시겠어요?" | "그때 어떻게 하셨어요?" |
| **유도하지 않는다** | "침대 못 구할까 봐 불안하셨죠?" | "하루 중 언제가 제일 신경 쓰였어요?" |
| **구체를 캔다** | "불편한 게 있었나요?" | "가장 최근에 그랬던 게 언제예요?" |
| **침묵을 견딘다** | 바로 다음 질문 | 3초 기다린다. 진짜 답은 그 다음에 나온다 |
| **제품을 설명하지 않는다** | "저희가 이런 걸 만드는데…" | 아무 말도 하지 않는다 |

> **"쓰시겠어요?"에 대한 대답은 전부 거짓말이다.** 사람들은 예의상 그렇다고 한다.
> 오직 **과거에 실제로 한 행동**만 데이터다.

### 3.6 질문지

#### 도입 (5분)

```
1. 언제, 어느 구간을 걸으셨어요?
2. 왜 카미노였어요?
   (→ 캐지 말 것. 대답이 짧으면 넘어간다)
3. 혼자 가셨어요, 같이 가셨어요?
```

#### 하루의 흐름 — H1, H2 검증 (12분)

```
4. 걷던 중 하루를 아침부터 저녁까지 그대로 얘기해주실 수 있을까요?
   몇 시에 일어나서 뭘 하셨는지.
   ★ 이 질문이 인터뷰의 핵심이다. 충분히 시간을 준다.

5. (기상 시각이 이르면) 왜 그 시간에 일어나셨어요?
   ★ H1: "침대" 이야기가 나오는지 본다. 우리가 먼저 꺼내지 않는다.

6. 하루 중 제일 신경 쓰였던 게 뭐예요?

7. 오늘 어디서 잘지는 언제 정하셨어요?
   - 전날 밤? 아침? 걷다가?
   - 정해놓고 걸으신 적과 안 정하고 걸으신 적, 어느 쪽이 많았어요?

8. 침대를 못 구하신 적 있어요?
   - (있으면) 몇 번이나? 그때 어떻게 하셨어요?
   - (없으면) 못 구할까 봐 걱정은 하셨어요?
   ★ H1의 핵심. "걱정은 했지만 실제로는 없었다"가 많이 나오면
     베드 레이스는 실재가 아니라 공포다 → 우리 가설이 맞다.

9. 걷는 동안 휴대폰을 몇 번쯤 꺼내셨어요? 주로 언제, 뭘 보려고?
   ★ H2. 여기서 "거의 안 봤다"가 다수면 앱 전략을 재검토해야 한다.
```


## 4. B · 웹 트래픽 실험

### 4.1 목적

**H5 검증**: 한국인이 카미노를 준비하며 실제로 검색하는가, 그리고 우리 페이지가 그 검색에 걸리는가.

도달률 전략의 60%가 검색인데, **이게 안 되면 전략 전체를 다시 짜야 한다.**

### 4.2 설계

| 항목 | 내용 |
|---|---|
| 기간 | **4주** (색인에 2주쯤 걸린다. 6주까지 볼 수도) |
| 시기 | 2026년 9~10월. **가을 시즌 준비자가 검색하는 때** |
| 범위 | 일정 계산기 + 마을 82 + 구간 81 + 루트 3 = 약 170페이지 |
| 홍보 | **하지 않는다.** 순수 검색 유입만 본다 |
| 비용 | 도메인·서버 약 30만 |

**⚠️ 카페에 홍보하고 싶은 유혹을 참는다.** 홍보하면 검색 수요를 측정할 수 없다.

### 4.3 측정

| 지표 | 도구 |
|---|---|
| 검색 노출 수 / 클릭 수 / 평균 순위 | Google Search Console, 네이버 서치어드바이저 |
| 유입 검색어 | 동일 |
| 페이지별 유입 | Vercel Analytics |
| 계산기 완료율 | 커스텀 이벤트 |
| 계획 링크 공유 수 | 커스텀 이벤트 |
| 체류 시간 | Analytics |

### 4.4 판단 기준 (사전에 정한다)

**이걸 미리 정해두지 않으면 결과를 보고 유리하게 해석하게 된다.**

| 결과 | 4주 후 검색 유입 | 판단 |
|---|---|---|
| 🟢 | **월 1,500 세션 이상** | 전략 유효. 계속 |
| 🟡 | 월 500~1,500 | 콘텐츠 부족. 6주 더 보고 재판단 |
| 🔴 | **월 500 미만** | **검색 전략 재검토.** 카페·유튜브 중심으로 전환 검토 |

**보조 지표**

- 계산기 완료율 40% 미만 → UX 문제
- 평균 체류 30초 미만 → 콘텐츠가 검색 의도와 안 맞음
- 네이버 유입이 구글의 1/3 미만 → **네이버 대응 별도 필요** (한국 시장에서 치명적)

### 4.5 ⚠️ 네이버를 과소평가하지 않는다

한국인 순례자는 **네이버에서 검색한다.** 그런데 네이버는 외부 사이트 색인이 구글만큼 관대하지 않다.

- 네이버 서치어드바이저 등록은 필수
- 네이버는 블로그·카페 콘텐츠를 우선 노출한다
- **자체 사이트만으로 네이버를 뚫기 어려울 수 있다** → 블로그 병행 운영 검토

**이건 실험으로만 알 수 있다. 4주 결과에서 네이버 비중을 반드시 본다.**

**목표: 약국 5곳, 알베르게 20곳에서 검증.**
**H20이 부정적이면 설계를 바꾼다** — 상대는 가리키기만 하고 내가 누르는 방식으로.

#### 관찰 기록 (매일)

```
· 알베르게 앞에 줄이 있었나? 몇 시부터?
· 만실이라 돌아가는 사람을 봤나?
· 한국인을 몇 명 만났나? 어떤 앱을 쓰던가?
· 오늘 스페인어가 필요했던 순간
· 내가 몰라서 곤란했던 것 ← 제품 아이디어의 원천
```

### 6.4 인터뷰를 길 위에서도 한다

**40일 동안 한국인 순례자를 계속 만난다.** 정식 인터뷰 20건과 별개로, 길에서 짧게 묻는다.

```
· 오늘 몇 시에 나오셨어요? 왜요?
· 숙소는 언제 정하세요?
· 뭐 보고 다니세요? (앱)
· 준비하면서 뭐가 제일 답답했어요?
```

**목표 30명.** 3장의 인터뷰보다 짧지만 표본이 커진다.

### 6.5 콘텐츠 산출

| 산출물 | 빈도 | 용도 |
|---|---|---|
| 일일 사진 + 짧은 글 | 매일 | 브런치·인스타 연재 |
| 영상 클립 | 격일 | 유튜브 |
| 구간별 정리 글 | 주 1회 | **SEO 가이드 페이지의 원료** |
| 숙소 사진 | 매일 | 제품 데이터 |

⚠️ **콘텐츠 제작이 조사를 방해하지 않게 한다.** 기록이 먼저다.

### 6.6 장비

| 항목 | 이유 |
|---|---|
| GPS 워치 또는 GPX 기록 앱 | 트랙 확보 |
| **보조 배터리 20,000mAh 이상** | 기록량이 많다 |
| 방수 폰 케이스 | 갈리시아는 비가 온다 |
| 노트 + 펜 | **배터리가 없을 때 유일한 수단** |
| 명함 또는 안내지 | 알베르게 주인에게 목적 설명용 (스페인어) |

### 6.7 ⚠️ 안전

**이건 조사이기 전에 800km를 걷는 일이다.**

- 목표 데이터를 채우려고 무리해서 걷지 않는다
- 몸이 아프면 **일정보다 몸이 우선**이다. 쉬거나 버스를 탄다
- 여행자 보험은 필수
- 가족에게 일정 공유
- 혼자라면 매일 생존 신고

> **역설적이지만, 우리가 만드는 제품의 핵심 메시지가 "무리하지 마라"다.**
> 창업자가 데이터 욕심에 무리하면 제품 철학과 모순된다.

## 9. 이 계획의 한계

정직하게 적는다.

| 한계 | 영향 |
|---|---|
| **20명은 통계적으로 유의하지 않다** | 정성 조사다. 경향만 본다. 수치화하지 않는다 |
| **모집 편향** | 카페·블로그에서 모으면 정보 탐색 적극층에 치우친다. **가장 도움이 필요한 무관심층이 빠진다** |
| **1인 실측의 주관성** | 체감 난이도, 통증은 개인차가 크다. H15 검증의 근거로는 약하다 |
| **가을 1회 관측** | 성수기(5~6월) 혼잡을 못 본다. **H1의 결정적 검증은 못 한다** |
| **트래픽 4주는 짧다** | SEO는 원래 수개월이 걸린다. 초기 신호로만 본다 |
| **갈림길은 한 번에 한쪽만 걷는다** | **11개 분기점마다 변형이 2~3개인데, 한 번의 도보로는 각 분기점당 하나만 실측된다.** 나머지 변형(예: 사모스를 걸으면 산 실은 못 걷는다)은 공개 GPX나 후속 방문으로 보완해야 한다 — M19(갈림길 추천) 완성이 그만큼 늦어질 수 있다 |

**→ 특히 마지막 두 개가 아프다.** 성수기 혼잡과 장기 SEO는 2027년 봄에야 확인된다. 그때까지는 잠정 판단으로 간다.
